# Current state

Each arvo crate has a `BACKLOG.md.tmpl` documenting deferred design
items. Entries often mention types (`UFixed<16, 0, Hot>`, `USize`,
`Bool`), but there's no explicit reminder keeping new entries
conformant with the type-surface rules. Future contributors (or
future-us) can quietly drift into `u32` / `bool` / `Option` shorthand
in promissory notes, which then leaks into implementations when the
promise is picked up.

# Changes proposed

Prepend a single compact reminder block to every BACKLOG.md.tmpl:

```markdown
> **Type-surface reminder**: new entries use `arvo::USize` /
> `Bool` / `UFixed<N, 0, S>` instead of `usize` / `bool` / `u*`;
> `notko::Maybe` / `Outcome` instead of `Option` / `Result`. See
> `rules/type-surface.md`.
```

The block goes between the top-level `# <crate> — backlog` heading
and the first existing paragraph. Nine arvo BACKLOG files touched.

Same block lands in hilavitkutin's six BACKLOG files in a parallel
hilavitkutin round (hilavitkutin adds `Str` to the list and drops
the crate-specific `arvo::` prefix where it's redundant).

# Non-goals

- No retroactive rewriting of existing BACKLOG entries — the block
  is a reminder, not an enforcement sweep.
- No lint change. Writing-style already catches the bigger drift;
  the reminder is agent/contributor guidance.

# Validation

- `cargo mock --lint-only` clean (no new writing-style hits; the
  block is blockquote prose, no em-dashes).
