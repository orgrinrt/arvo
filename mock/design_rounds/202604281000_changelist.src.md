## Pass E — Strategy-axis-driven arithmetic dispatch

This is the largest pass. Recommend splitting into per-trait commits.

`mock/crates/arvo/src/traits.rs`:
- `UArith<BITS>` and `IArith<BITS>`: refactor bound from `S: Strategy + UContainerFor<BITS>` to `S: HasAxes<Overflow = O, Container = C, Layout = L>` where the impl is parameterised over the axis projections. Concrete bodies dispatch on `O::DISCRIMINANT` etc. in const branches that monomorphise.
- `UWidenFrom` / `UNarrowFrom`: same shape. The cross-strategy resolution still goes through `Resolve<S1, S2>::Out`, but the per-S impl now bounds on axes.

`mock/crates/arvo/src/ufixed_impl.rs` and `ifixed_impl.rs`:
- Update the `impl ... for UFixed<I, F, S> where S: ...` blocks to bound on the axis traits where the body needs them.

`mock/crates/arvo-strategy/src/strategy.rs`:
- Ensure `Hot`, `Warm`, `Cold`, `Precise` each implement `HasAxes` with the axis-marker projections matching the documented table.

The consumer call sites do not change. The `S: Strategy` bound at the consumer-facing public API stays; the implementation-side dispatch goes through axes internally.

## Pass G — Cross-strategy resolution warning

`mock/crates/arvo-strategy/src/cross_strategy.rs` (new):
- `pub trait CrossStrategyOp<S1: Strategy, S2: Strategy>` with `#[diagnostic::on_unimplemented]`.
- Implement for `(S, S)` pairs (same-strategy ops are silently fine).
- Do NOT implement for distinct-strategy pairs; the absence triggers the diagnostic at the call site when the consumer composes a cross-strategy op.

`mock/crates/arvo/src/traits.rs`:
- Add `where Self::Lhs: CrossStrategyOp<S1, S2>` (or similar) at the appropriate trait bound positions on cross-strategy arithmetic methods.

`mock/crates/arvo-strategy/src/strategy.rs`:
- Re-export `CrossStrategyOp` if needed.

Iterate on the diagnostic text. Initial draft: "cross-strategy op `{Self}` mixes `{S1}` and `{S2}`; the resolved strategy `{Out}` adopted `{Out's overflow policy}`; insert an explicit `.cast::<{Out}>()` to silence this warning, or annotate with `#[arvo::lint:allow(cross-strategy-resolution)]`".
