## Open question (blocks src-CL lock)

### Float widths

The doc CL says "wraps `f32`/`f64`" without picking. Options:

- **(a)** Two pairs: `FastFloat32(f32)` + `FastFloat64(f64)` and `StrictFloat32` + `StrictFloat64`. `Float32` / `Float64` cfg-aliases. Explicit, no accidental precision drift. More types.
- **(b)** Single width chosen now (likely `f32` for SIMD-friendly default). Minimal surface. Locked into one width until a future round lifts it.
- **(c)** Generic `FastFloat<F: Ieee>` / `StrictFloat<F: Ieee>` where `Ieee` is a sealed trait impl'd for `f32` + `f64`. Zero-cost, flexible, slightly heavier API.

My recommendation: **(c)**. Matches the strategy-generic pattern we already use for `UFixed`/`IFixed`, keeps the API shape uniform, and the sealed trait stops arbitrary types from sneaking in. `Float` alias resolves with a concrete float width chosen by consumer or by `cfg(arvo_fast_math)`.

Override with (a) or (b) if you prefer. This is the only blocker before src-CL lock.

## Impl-time decisions

These are not blockers; I'll pick at impl time unless the user objects.

1. **Warm const-bound mechanism:** `where [(); (I.0 as usize + F.0 as usize) * (if I.0 + F.0 <= 32 { 1 } else { 0 }) == (I.0 as usize + F.0 as usize)]:` ‚Äî the "identity when valid, mismatch when invalid" pattern, most portable on current nightly. Alternative (sealed trait `WarmCompatible`) if the above fights with the const evaluator.
2. **Container types:** `Strategy::Container<const I: IBits, const F: FBits>` associated type. Each marker impls with concrete `u8`/`u16`/`u32`/`u64` via `const fn`-based selection. `typenum`-style tables if const-fn bounds get gnarly.
3. **`repr(transparent)`** on every numeric type. Verified via `size_of` unit tests.
4. **Strategy conversion** as `From`/`TryFrom` impls between matching `<I, F>` tuples across strategies (free for widen, fallible for narrow).
5. **Cross-width ops** via blanket impls of `Add<UFixed<I2, F2, S2>>` for `UFixed<I1, F1, S1>` ‚Äî return type has `I = max(I1, I2)`, `F = max(F1, F2)`, `S` = more conservative.
