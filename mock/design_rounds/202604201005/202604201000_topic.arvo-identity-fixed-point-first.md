## What this round is

A documentation-quality failure: arvo's current `mock/PRINCIPLES.md.tmpl`
+ `mock/crates/arvo/DESIGN.md.tmpl` carry the forbidden/use table
("don't use `u8`, do use `UFixed<I, 0, S>`") but LOST arvo's identity
as a fixed-point arithmetic stack. The originals in
`polka-dots/mock/` and `saalis/mock/` carry the identity explicitly.
Current docs only carry the *rule form*, and rule-form without
identity-form doesn't stick — agents (including the human authors and overseers) kept
reaching for raw `u64` / `u32` / `bool` despite multiple passes over
the docs.

This round is doc-only: no source changes. Port the identity framing
back into the arvo templates so a reader understands what arvo IS
before they encounter the rules about what arvo FORBIDS.

## The identity statement

Convergent from the originals, in one sentence:

> **arvo is a fixed-point arithmetic stack: the exclusive numeric
> substrate for the clause-dev workspace (hilavitkutin, polka-dots,
> saalis, loimu). Every numeric primitive flows through
> `UFixed` / `IFixed` / `FastFloat` / `StrictFloat`, never through
> raw `u8` / `u16` / `u32` / `u64` / `u128` / `i8` / ... / `f32` /
> `f64` / `bool`, enforced by the `arvo-types-only` lint.**

This isn't a new rule. It's a restoration.
