## Impl-time decisions (pre-committed)

1. **Float typing (option c).** Sealed `Ieee` trait with private supertrait pattern:
   ```rust
   mod sealed { pub trait Sealed {} impl Sealed for f32 {} impl Sealed for f64 {} }
   pub trait Ieee: sealed::Sealed + Copy { /* assoc types, width, etc. */ }
   impl Ieee for f32 { ... } impl Ieee for f64 { ... }
   pub struct FastFloat<F: Ieee>(F);
   pub struct StrictFloat<F: Ieee>(F);
   #[cfg(arvo_fast_math)] pub type Float<F: Ieee = f32> = FastFloat<F>;
   #[cfg(not(arvo_fast_math))] pub type Float<F: Ieee = f32> = StrictFloat<F>;
   ```
   The sealed supertrait keeps arbitrary types out while letting consumers pick `f32` or `f64` explicitly.

2. **Warm const-bound mechanism.** Primary: `where [(); (I.0 as usize + F.0 as usize) * (if I.0 + F.0 <= 32 { 1 } else { 0 }) == (I.0 as usize + F.0 as usize)]:` — the identity-when-valid trick, most portable on current nightly. Fallback if const evaluator fights: sealed `WarmCompatible<const I: IBits, const F: FBits>` trait with blanket impls only for the valid (I, F) pairs.

3. **Container types.** `Strategy::Container<const I: IBits, const F: FBits>` associated type; each marker impls with concrete integer types via `const fn` selection. If const-fn bound matching gets gnarly, use the sealed-trait table pattern (one impl per valid (I, F) shape).

4. **`repr(transparent)`** on every numeric wrapper. Verified via `size_of` unit tests in the test suite.

5. **Strategy conversion.** `From`/`TryFrom` between matching `<I, F>` tuples across strategies — `From` for widen-free (Hot→Warm, Warm→Precise…), `TryFrom` for lossy narrows.

6. **Cross-width ops.** Blanket impls of `Add<UFixed<I2, F2, S2>> for UFixed<I1, F1, S1>` — return type has `I = max(I1, I2)`, `F = max(F1, F2)`, `S` = more conservative of `{S1, S2}` per the doc CL's strategy-resolution rule.

## Plan of attack (dependency order)

1. `newtype.rs` — stand-alone; only `core` deps.
2. `strategy.rs` — stand-alone.
3. `markers.rs` — depends on newtypes + strategy.
4. `ufixed.rs`, `ifixed.rs` — depends on newtypes + strategy + markers.
5. `float.rs` — stand-alone (sealed Ieee trait + FastFloat / StrictFloat / Float).
6. `traits.rs` — depends on numerics for blanket impls.
7. Tests (including negative compile test).
8. `cargo mock` regenerates `docs/` and `.claude/` from templates.

Dispatched as one impl subagent once the src CL is locked. Subagent follows the locked CL mechanically; if anything is underspecified, it pauses and escalates rather than guessing.
