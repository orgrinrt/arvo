The `Bits` struct definition:

> `Bits<const N: u8, S: Strategy>` is the L0 storage primitive. It moved to arvo from arvo-bits in round 202604500000 because every meta-layer ConstParamTy newtype (`IBits`, `FBits`, `Width`) wraps `Bits<7, Hot>`, and `UFixed` / `IFixed` use `Bits<{ufixed_bits(I, F)}, S>` for transparent storage. Hosting `Bits` in arvo-bits would create a circular dependency with arvo's own primitive layer.
>
> `Bits` is opaque to arithmetic (no `Add` / `Sub` / `Mul` / etc.) by deliberate design. It is the storage carrier; arithmetic types (`UFixed`, `IFixed`) wrap it and impl arithmetic at their layer.

Document the constructor surface (carried over from §H-1 of the topic):

```rust
impl<const N: u8, S: Strategy> Bits<N, S>
where Hot: UContainerFor<N>, S: UContainerFor<N>
{
    /// Associated-type-typed constructor.
    pub const fn from_raw(raw: <S as UContainerFor<N>>::T) -> Self;

    /// Universal masking entry (Hot-container width). Masks input to N bits.
    pub const fn from_raw_u64(raw: u64) -> Self;

    /// Width-class siblings. Each masks within its width class.
    pub const fn from_raw_u32(raw: u32) -> Self;
    pub const fn from_raw_u16(raw: u16) -> Self;
    pub const fn from_raw_u8(raw: u8) -> Self;

    /// Existing accessor.
    pub const fn to_raw(self) -> <S as UContainerFor<N>>::T;

    /// Symmetric meta-layer accessor (reads `Width.0.0` etc.).
    pub const fn to_raw_u8(self) -> u8;
}
```

Consumers migrate `Bits::<N, S>::new(x)` → `Bits::<N, S>::from_raw_u64(x)` and `b.bits()` → `b.to_raw_u64()` (or the width-class equivalent).

Document soundness of `from_raw_uN` masking (carried over from §H-2 of the topic):
