## Round B shipping scope (doc-side)

The doc CL describes the round's full design intent:

1. Sign axis on Bits (storage primitive gains `Sign: Signedness` const-generic).
2. Meta-newtype carrier widening (IBits / FBits / Width carry `Bits<9, Hot>`, range 0..=256).
3. UContainerFor / IContainerFor extension to 65..=128 via native `u128` / `i128`.
4. `MultiContainer<HiT, LoT>` storage primitive declared in arvo-storage.
5. Bits at 129..=256 via MultiContainer projection (storage-only; arithmetic deferred).
6. IFixed reshape: `repr(transparent)` over `Bits<{1+I+F}, S, Signed>` (closes pass 4d).

Round B's arithmetic surface for multi-value containers (UArith / IArith dispatch on MultiContainer, UWidenFrom / UNarrowFrom cross-container widening) is BACKLOG-tracked for a Round B-arith follow-up. The storage shape commits in Round B; UFixed / IFixed at 129..=256 logical width require the arith bridge before they compile, so their cap stays at 128 in this round (with `Bits<129..=256, S, Sign>` available standalone). This staging keeps Round B's commit count manageable while landing the foundational shape changes.
