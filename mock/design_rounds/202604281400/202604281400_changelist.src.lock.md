## SRC.1. Em-dash sweep in `arvo-storage/src/layout_assertions.rs`

`mock/crates/arvo-storage/src/layout_assertions.rs`
- Five section-divider comment lines in the Tier 2 Bits projection block
  contain em-dash tokens. Replace each em-dash with a colon.
- Affected lines (current state):
  - Line 63: `// --- Tier 2: Bits projection — Hot, unsigned ------------------------------`
  - Line 80: `// --- Tier 2: Bits projection — Hot, signed --------------------------------`
  - Line 97: `// --- Tier 2: Bits projection — Cold mirrors Hot ---------------------------`
  - Line 113: `// --- Tier 2: Bits projection — Warm (2x logical, caps at 64) --------------`
  - Line 130: `// --- Tier 2: Bits projection — Precise ------------------------------------`
- New form replaces the em-dash with a colon and a space, preserving the
  surrounding `--- Tier 2: Bits projection ` prefix and `---` trailing
  rule. The visual divider intent stays; only the em-dash becomes a colon.
  Example: `// --- Tier 2: Bits projection: Hot, unsigned ------------------------------`.

The `writing-style.md` workspace rule is hard-zero on em-dashes anywhere
including inline comments. The pre-commit gate will treat the new form as
clean.

## SRC.2. `arvo-storage/Cargo.toml` check-cfg registration

`mock/crates/arvo-storage/Cargo.toml`
- After the `[features]` section, add:
  ```toml
  [lints.rust]
  unexpected_cfgs = { level = "warn", check-cfg = ["cfg(arvo_full_layout_audit)"] }
  ```
- The cfg name is the runtime-shape "audit-only" framing introduced by
  Round D Pass A.2. Registering it in `check-cfg` silences the
  `unexpected_cfgs` warning on every workspace `cargo check` without
  changing the cfg's semantics.
- Per-crate lints config is the right shape (the cfg is only consumed in
  this crate's `layout_assertions.rs`); no workspace-level change needed.

## SRC.3. Drop unused nightly-feature decls in `arvo-narrow/src/lib.rs`

`mock/crates/arvo-narrow/src/lib.rs`
- Remove the three lines:
  ```rust
  #![feature(adt_const_params)]
  #![feature(const_trait_impl)]
  #![feature(const_param_ty_trait)]
  ```
- These were load-bearing pre-Pass-B.4 when `arvo-narrow` hosted the
  cross-primitive Narrow impls. Pass B.4 relocated those impls into
  `arvo-bits-contracts` (orphan rules forced the move) and arvo-narrow is
  now a re-export facade. The features are dead.
- Keep any other crate-level attributes intact (`#![no_std]`, doc
  comments, the `pub use arvo_bits_contracts::{Narrow, Narrowed};`
  re-export).
- `#![feature(generic_const_exprs)]`, if also present, may be inspectable
  for removal at the same time. Verify the feature is unused by the
  re-export; if so, remove it. If a doc comment in this file references
  any of the removed features, drop the reference.
