This doc CL operationalises the frozen topic. It enumerates every doc-template edit required to reach the target architecture: arvo as downstream facade, a new minimal L0 crate, six trait-contract crates, the existing implementation crates pinned to their post-restructure roles, the const-trait migration sweep, and the ergonomic `Int<N, S>` / `Uint<N, S>` aliases. Source-side mutation lands in the subsequent SRC CL.

The work covers seven concern groups:

1. New L0 crate creation: `arvo-storage` (working name, see L0-1) hosting `Bits<N, S>`, strategy markers, `Bool`, `USize`, and the typed meta-bit newtypes (`IBits`, `FBits`, `Width`).
2. Six trait-contract crate creations: `arvo-transparent`, `arvo-strategy`, `arvo-bits-contracts`, `arvo-mask-contracts`, `arvo-narrow-contracts`, `arvo-numeric-contracts`. Each pulls only what its consumers need.
3. arvo facade reshape: re-exports surface, hosts default impls, defines `Int<N, S>` / `Uint<N, S>` and the eight surviving canonical aliases, hosts `UFixed` / `IFixed` as `repr(transparent)` newtypes over `Bits`.
4. Implementation crate updates: `arvo-bits` (removes hosted `Bits` storage, retains contract impls on the new L0 type), `arvo-bitmask` (becomes home of `Narrow<T>` impls), `arvo-narrow` (new crate for narrow contract impls per Q-C), `arvo-hash` (Hasher becomes const trait).
5. Const-trait migration: every trait listed in D-12 becomes `pub const trait` in its contract crate.
6. Per-size helper deletion plan from `Bits` and bitfield-macro rewrite.
7. Q-D alias deletion plan: 254 `Uint*`/`Int*` aliases collapse to 8 canonical std-parallel survivors plus the generic forms.

## Bitfield-macro rewrite spec (D-10)

The `bitfield!` proc macro keeps its DSL surface (declarative struct + slot list with bit ranges + types). What changes is the emitted body shape.

Before this round, expansions produced inline `Bits::<N, Hot>::new(u64)` / `Bits::<...>::bits()` / hand-rolled mask + shift arithmetic per slot.

After this round, expansions produce:

- `Slot<Parent, const LO: u8, const HI: u8, T>` getters that delegate to `Narrow<T>::narrow_to::<{HI - LO + 1}>(self.0.bits_in_range::<LO, HI>())` (or the equivalent const-trait path; exact spelling depends on the `Narrow` / `BitSequence` final surface).
- Setters that compose `Mask<W>::mask_for_range::<LO, HI>()` with `BitLogic::and` / `BitLogic::or` and `BitLogic::shl` / `BitLogic::shr` to write the slot in place.
- No inline u64 arithmetic. Every operation goes through a const trait method.
- No per-N expansion bloat: the macro emits one Slot generic per slot, not a width-specialised expansion.

The macro source crate (`arvo-macros` or wherever it currently lives. SRC CL confirms) gets its emitted-body builder rewritten to compose const-trait method calls. The DSL parser is unchanged.

## Q-D alias deletion plan

Every `Uint*` and `Int*` alias other than the eight std-parallel canonical survivors gets deleted from the arvo facade. Concrete deletion list (search-and-replace across DESIGN templates + SRC):

Surviving (eight aliases): `Uint8`, `Uint16`, `Uint32`, `Uint64`, `Int8`, `Int16`, `Int32`, `Int64`.

Deleted (every other width. partial enumeration; SRC CL ships the full list from the existing per-N alias macros):
- `Uint1`, `Uint2`, `Uint3`, `Uint4`, `Uint5`, `Uint6`, `Uint7`
- `Uint9`, `Uint10`, `Uint11`, `Uint12`, `Uint13`, `Uint14`, `Uint15`
- `Uint17` through `Uint31`
- `Uint33` through `Uint63`
- All `Int*` widths matching the same pattern (`Int1`..`Int7`, `Int9`..`Int15`, `Int17`..`Int31`, `Int33`..`Int63`)
- Any wider `Uint*` / `Int*` that may exist at higher N (out of `<= 64` band, currently not implemented; deletion is moot but SRC removes the generation if any macro produces them)

Consumer migration: every existing `Uint13` / `Int7` reference in arvo crates and downstream consumer crates rewrites to the generic form `Uint<13>` / `Int<7>`. The SRC CL ships the per-call-site rewrite table.
