## Deliverables

- `Matrix<W: Copy, const N: Cap>` — dense `N × N` matrix wrapper with private `[[W; _]; _]` storage. Public surface: `new`, `from_fn`, `get(i: USize, j: USize) -> W`, `set(&mut, i, j, W)`, `diagonal() -> [W; _]`. `repr(transparent)` over the inner array-of-arrays.
- `fn laplacian<const N: Cap, W, F>(weights: &Matrix<W, N>) -> Matrix<F, N>` — builds `L = D - W` where `D` is the diagonal degree matrix. Uses `Into<F>` for weight→float conversion at the boundary.
- `fn power_iteration<const N: Cap, F>(matrix: &Matrix<F, N>, iterations: USize) -> [F; N.0.0 as usize]` — dominant eigenvector via repeated matrix-vector multiply and normalisation.
- `fn fiedler_vector<const N: Cap, W, F>(weights: &Matrix<W, N>, iterations: USize) -> [F; N.0.0 as usize]` — power iteration on the deflated Laplacian.
- `fn spectral_bisection<const N: Cap, F>(fiedler: &[F; N.0.0 as usize]) -> (Mask64, Mask64)` — sign-based two-way partition; requires `N ≤ 64` (Mask64 output).
- `fn k_way_partition<const N: Cap, const K: Cap, W, F>(weights: &Matrix<W, N>, iterations: USize) -> (USize, [USize; N.0.0 as usize])` — recursive spectral bisection.
- Unit tests: Matrix round-trip, Laplacian of known-weighted graph, power iteration convergence on a known-eigenvector matrix, Fiedler sign correctness on a known-bipartition graph, spectral_bisection classification, k_way with K=4 on a known 4-cluster graph.

- Promotion of `Matrix<W, N>` to arvo-sparse (backlog).
- Non-symmetric matrices.

## Impl-time decisions (pre-committed)

1. **`Matrix<W, const N: Cap>`** stores `[[W; N.0.0 as usize]; N.0.0 as usize]`. The `.0.0` double-unwrap (Cap → USize → usize) is the L0-established pattern for carrying a newtype at the const-generic param surface while reaching raw `usize` for array sizing. Public methods accept `i: USize, j: USize` and extract via `.0` for array indexing.

2. **`Matrix::new` vs `Matrix::from_fn`.** `new` takes the full inner array (for consumer already holding one). `from_fn` constructs row-by-row via the closure — cleaner for test setup and for `Matrix::identity` / `Matrix::diagonal_from_vector` patterns if they surface later.

3. **`laplacian`**: weighted Laplacian `L[i][i] = Σ w[i][j] for j != i` (degree), `L[i][j] = -w[i][j] for i != j`. Conversion `W → F` via `Into<F>` happens per element.

4. **`power_iteration`** normalises the vector by its L2 norm after each multiply. Uses `Sqrt` on `F`. Terminates at `iterations` count (no convergence-ratio check this round — benchmark concern).

5. **`fiedler_vector`** deflates by subtracting the projection onto the all-ones eigenvector: `v = v - (v·1/N)·1` where 1 is the all-ones vector. Then power iteration.

6. **`spectral_bisection`** assigns node `i` to positive-mask if `fiedler[i] > 0`, negative-mask otherwise (ties go negative). Returns `(positive_mask, negative_mask)` as `(Mask64, Mask64)`. Caller provides `N ≤ 64`.

7. **`k_way_partition`** recursive bisection. Maintain a stack of (node-set-mask, partition-id-floor) and partition-id counter. When a set has size ≥ 2 and we still have budget in K, bisect it; else fix its partition id. Returns final `(partition_count, [partition_id_per_node])`.

8. **`Cap` usage.** Tests pass explicit `Cap(USize(N))` const values. Uses short helper constants like `const C4: Cap = Cap(USize(4));` at test-file scope when handy.

9. **No `#[inline]`** this round.
