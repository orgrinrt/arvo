# Src Changelist: arvo-bitmask Cap/USize cleanup

**Date:** 2026-04-26
**Phase:** SRC
**Scope:** `arvo-bitmask` crate only.
**Doc CL:** 202604260000_changelist.doc.lock.md

## Deliverables

- Introduce `pub const fn cap_size(c: Cap) -> usize` at the crate
  root (re-exported from `mod matrix`). Unwraps `Cap(USize(usize))`
  to `usize` for array-sizing in `generic_const_exprs` contexts
  where inline `.0.0` is rejected.
- `BitMatrix64<const N: usize>` → `BitMatrix64<const N: Cap>` across
  the struct, all impl blocks, `Default`, and `propagate_dirty_64`.
  Storage goes from `[Mask64; N]` to `[Mask64; cap_size(N)]` with
  `where [(); cap_size(N)]:` bound on every impl that touches
  storage or indexing.
- Same for `BitMatrix256<const N: Cap>` and `propagate_dirty_256`.
- Remove the incorrect `//! Const-generic N: usize rather than
  arvo::Cap` rationale from the `matrix.rs` header comment. Replace
  with a short description of the `cap_size` pattern.
- Update the 14 existing `matrix.rs` unit tests + the 14
  `tests/matrix.rs` integration tests + the 4 `tests/dirty.rs`
  integration tests to construct `Cap(USize(N))` values at call
  sites. Add a `const fn cap(n: usize) -> Cap` test helper at
  test-file scope where it reduces noise.
- No change to: `Mask64` / `Mask256` / bit-scanning / set-ops APIs
  (not parameterised by `Cap`); `NodeId` (already `USize`);
  `dirty.rs` function bodies (algorithm unchanged).

Out of scope (per doc CL):
- arvo / arvo-bits / arvo-graph / arvo-sparse / arvo-comb / arvo-spectral src.
- Promoting `cap_size` into arvo L0 (backlog).
- `BitMatrix` trait abstraction over Mask64/Mask256 (backlog).

## Impl-time decisions (pre-committed)

1. **`cap_size` placement.** Define as `pub const fn cap_size(c:
   Cap) -> usize { c.0.0 }` at the top of `matrix.rs` (module
   scope), then re-export from `lib.rs` as `pub use
   matrix::cap_size;`. This mirrors arvo-spectral's placement and
   keeps it accessible to consumer code that also uses `const N:
   Cap` without importing from a deeper path.

2. **`Cap` import.** `matrix.rs` already uses `arvo::newtype::{Bool,
   USize}`; add `Cap` to the import list. `dirty.rs` adds the same.

3. **Storage reshape.** `[Mask64; N]` becomes `[Mask64; cap_size(N)]`
   with `where [(); cap_size(N)]:` on every `impl<const N: Cap>`
   block that uses the storage. The bound is redundant on some impl
   blocks but required on any that constructs / iterates the array.
   Apply to all blocks uniformly for consistency.

4. **Default impl.** `BitMatrix64<N>::default()` currently returns
   `Self { rows: [Mask64::default(); N] }`. Under `Cap`, this becomes
   `Self { rows: [Mask64::default(); cap_size(N)] }` with the
   `[(); cap_size(N)]:` bound on the Default impl.

5. **Method bodies using `N` as bound.** Any loop `for i in 0..N`
   becomes `for i in 0..cap_size(N)`. Internal `usize` index
   variables stay internal — the public API method signatures
   already take `NodeId` or `USize`, so no further surface change.

6. **Test construction helper.** In test modules:
   ```rust
   const fn cap(n: usize) -> Cap { Cap(USize(n)) }
   const C4: Cap = cap(4);
   const C8: Cap = cap(8);
   ```
   Used wherever tests instantiate `BitMatrix64<_>` or
   `BitMatrix256<_>`. Keeps test bodies readable.

7. **No `#[inline]` added.** Performance characteristics unchanged
   since the helper const-fn evaluates at compile time.

8. **No BACKLOG-to-SHAME conversions.** The new BACKLOG entries
   (cap_size promotion, BitMatrix trait abstraction, BitMatrix iter
   contract) are design-future items, not current gaps that need a
   SHAME escape-hatch.
