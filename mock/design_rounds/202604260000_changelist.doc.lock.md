# Doc Changelist: arvo L2 Cap/USize cascade cleanup

**Date:** 2026-04-26
**Phase:** DOC
**Scope:** `arvo-bitmask`, `arvo-graph`, `arvo-sparse` crates.
**Source topics:**
- 202603241400_topic.arvo-crate-dependency-revision.md (generic contracts)

## Purpose

Retroactive cleanup: the initial arvo-bitmask / arvo-graph /
arvo-sparse impl rounds used `const N: usize` on public surfaces
against each crate's DESIGN.md.tmpl, which calls for `const N: Cap`.
This round aligns the src with the design — mirrors the correction
made in arvo-spectral (round 202604250000) but applied retroactively
to the three L2 crates that were locked in first.

Scope note: the three crates cascade together because arvo-graph
and arvo-sparse take `BitMatrix64<const N: _>` in their own
function signatures — changing arvo-bitmask's const-generic alone
breaks downstream compilation, so a single round covers the chain.
`arvo-comb` does not consume `BitMatrix*` so it gets its own round.

Two corrections per crate:

1. **`const N: Cap`, not `const N: usize`.** Every public type and
   function signature takes the arvo `Cap` newtype as its
   const-generic size parameter. Array-sizing inside struct fields
   / method bodies extracts via `cap_size(c: Cap) -> usize { c.0.0 }`
   — inline `.0.0` is rejected by nightly `generic_const_exprs`.

2. **`USize` over `usize`** in count / index positions on the public
   API. Internal loop indices may stay raw `usize`; only method
   signatures surface the newtype.

`cap_size` lives in arvo-bitmask (`matrix::cap_size`) and is
re-exported from its crate root. arvo-graph and arvo-sparse import
it rather than redefining. A third consumer would trigger the
backlog-tracked promotion to arvo L0.

No algorithmic changes across the three crates. Storage layouts and
test semantics unchanged.

## Per-crate doc changes

### arvo-bitmask — `mock/crates/arvo-bitmask/DESIGN.md.tmpl`

- Pluralise the bit-matrix type entry from `BitMatrix<const N: Cap>`
  to `BitMatrix64<const N: Cap>` and `BitMatrix256<const N: Cap>` —
  matches concrete types shipped in src.
- Swap `propagate_dirty(dag: &BitMatrix, dirty: &mut Mask)` prose
  for the two concrete signatures `propagate_dirty_64` /
  `propagate_dirty_256`, both taking `const N: Cap`.

### arvo-bitmask — `mock/crates/arvo-bitmask/BACKLOG.md.tmpl` (new)

- `cap_size` helper promotion to arvo (L0).
- `BitMatrix` trait abstraction over Mask64/Mask256 (deferred; nightly
  generic-const-exprs fights it).
- `BitMatrix` contract in arvo-bits for row-access algorithms
  (deferred; no second consumer shape yet).

### arvo-graph — `mock/crates/arvo-graph/DESIGN.md.tmpl`

- Every `const N: usize` in public signatures (`topo_sort`,
  `rank_of`, `weakly_connected_components`, `shortest_path`,
  `minimum_spanning_tree`, `waist_nodes`, etc.) → `const N: Cap`.
- Array returns shown as `[USize; N]` become `[USize; cap_size(N)]`
  with the implied `where [(); cap_size(N)]:` bound (trait-level
  only; DESIGN doesn't spell the where-clause explicitly).
- Document the `cap_size` import from arvo-bitmask.

### arvo-sparse — `mock/crates/arvo-sparse/DESIGN.md.tmpl`

- Same treatment: every `const N: usize` / `const ROWS: usize` /
  `const NNZ: usize` in public signatures (`rcm_reorder_64`,
  `rcm_reorder_256`, `block_diagonal_64`, `block_diagonal_256`,
  `dulmage_mendelsohn_64`, `dulmage_mendelsohn_256`, the `Csr`
  type) → `const _: Cap`.
- `Csr<const ROWS: Cap, const NNZ: Cap, W>` becomes the new signature.
- Document `cap_size` import.

### arvo-graph, arvo-sparse — `BACKLOG.md.tmpl` (new entries)

- Each gets a short note that `cap_size` currently imports from
  arvo-bitmask and tracks the promotion to arvo (L0) when a third
  user surfaces.

## Files NOT changed

- arvo / arvo-bits / arvo-comb / arvo-spectral source. arvo-spectral
  is already Cap/USize-correct. arvo-comb gets its own cleanup round
  (does not consume `BitMatrix*`, independent scope). arvo / arvo-bits
  have no `const N: usize` on the L0 / L1 surface — arvo's `UFixed<I,
  F>` uses `IBits` / `FBits` newtypes already.
- Lints, hooks, agent templates. Task #50 tracks the generic
  newtype-at-boundaries lint that would have caught this earlier.
- Mockspace lint-rules.

## Comparison to deprecated changelist

This is the second authoring of this round's doc CL. The first
(locked briefly as `202604260000_changelist.doc.lock.md`) scoped to
arvo-bitmask only. Mid-impl it became clear that downstream
consumers (arvo-graph, arvo-sparse) take `BitMatrix64<const N:
usize>` and would not compile with the narrower scope. Unlocked via
`cargo mock unlock`, doc re-authored with the expanded cascade
scope, old src CL deprecated
(`202604260100_changelist.src.deprecated.md`).

No relation to the closed 202604210000 arvo-bitmask impl round —
that one shipped with `const N: usize` as originally agreed and is
archived under `design_rounds/202604210000/`. This round retroactively
aligns its src with DESIGN.
