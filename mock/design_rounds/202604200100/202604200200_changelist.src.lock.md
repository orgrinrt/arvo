## Impl-time decisions (pre-committed)

1. **Sealed BitPrim for bit ops.** Sealed via private supertrait; impl'd for `u8` / `u16` / `u32` / `u64` with `count_ones`, `trailing_zeros`, `leading_zeros`, `bit(self, idx) -> bool`, `with_bit_set(self, idx) -> Self`, `with_bit_cleared`, `with_bit_toggled`. Methods are `const fn` where possible (`wrapping_shl` etc.); otherwise plain inherent delegation.

2. **IBitPrim mirror.** `i8` / `i16` / `i32` / `i64` with the same surface. Bit operations reinterpret via `as u*` / `as i*` casts where necessary so signed containers can participate in bit-level manipulation without signed-shift surprise.

3. **Bounds on the concrete impls.** `impl<const I: IBits, const F: FBits, S: Strategy> BitAccess for UFixed<I, F, S> where S: UContainerFor<{ufixed_bits(I, F)}>, <S as UContainerFor<{ufixed_bits(I, F)}>>::T: BitPrim`. The associated-type bound keeps the container's primitive-integer requirement explicit and propagates cleanly through where-clauses. Mirrored for IFixed with `IContainerFor` / `IBitPrim`.

4. **Alias-width constants.** `IBits::FOUR` / `EIGHT` / `SIXTEEN` / `THIRTYTWO` / `SIXTYFOUR` defined as `pub(crate) const IBits_X: IBits = IBits(X);` module constants in arvo-bits `alias.rs`, spelled `IBits_EIGHT` etc. in code. Arvo L0 already provides `IBits::ZERO` / `ONE`. The alias type definitions reference these private constants.

5. **`is_zero` returns `Bool`.** Per DESIGN. Uses the `Bool` type from arvo (already exported via the L0 public API).

6. **`bit()` returns `Bool`.** Per DESIGN. `fn bit(self, idx: USize) -> Bool`.

7. **Index handling.** `idx >= WIDTH` is defined as returning `Bool::FALSE` for `bit()` and self-unchanged for `with_bit_*` (non-panicking out-of-range semantics). Rationale: easier for generic code to handle overflow without peppering length-preconditions.
