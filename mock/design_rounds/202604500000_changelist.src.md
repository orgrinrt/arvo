## Ergonomic surface for arvo primitive newtypes

Round-level addition (recorded 2026-04-26 mid-SRC after the IBits / FBits / USize field-type-change touch-set scope debate). The doc CL specifies the field-type changes (`IBits(pub Bits<7, Hot>)` etc.) but does not specify the ergonomic surface that makes the wrapping invisible at call sites. Without that surface, any field-type change cascades into a literal-construction sweep across every consumer call site (`IBits(7)` → can no longer construct directly because the field type is now `Bits<7, Hot>`, not `u8`). With the surface, the wrapping is transparent: `*ibits` reads the inner native, `IBits::from(7u8)` constructs, `ibits(7)` const-constructs in const-generic position, etc.

The pattern is established for `Bool` (Deref / Try / AsBool / `From<bool>` / `From<Bool> for bool`) and partially for `USize` (Deref). This section formalises the same pattern for every wrapper introduced or migrated by this round and ensures the existing wrappers carry the full set.

### The wrapper template

For every wrapper newtype `pub struct $W(pub $Inner)` (or its migrated form) where `$Inner` is itself an arvo primitive (`Bits<N, S>`, `usize` at platform-bound sites, etc.), implement:

1. **`Deref<Target = $Native>`** where `$Native` is the bare primitive that the wrapping ultimately bottoms out at (`u8` for `IBits` / `FBits` / `Width`; `usize` for `USize`; `bool` for `Bool`). The `deref` body routes through whatever wrap chain the field carries, returning a reference to the inner native. For `IBits(Bits<7, Hot>)` this means dereffing through `Bits`'s underlying container reference; for `USize(Bits<{cfg-platform-width}, Hot>)` the unsafe-transmute approach in the doc CL applies (the Bits's container IS layout-equivalent to `usize`).
2. **`From<$Native> for $W`** so call sites can construct from a raw literal: `let i: IBits = 7u8.into();` or `IBits::from(7u8)`. The body wraps via the const-fn helper (`ibits(n)`) or the equivalent.
3. **`From<$W> for $Native`** for the symmetric direction: `let n: u8 = i.into();`.
4. **`AsRef<$Native>`** for the borrow-only path (mirrors `Deref` but is callable as `i.as_ref()` without prefix `*`).
5. **Const-fn helper** at module scope with the lowercase name: `pub const fn ibits(n: u8) -> IBits` (already present), `pub const fn fbits(n: u8) -> FBits`, `pub const fn width(n: u8) -> Width`. Used in const-generic position where `From::from` is not const-callable.
6. **`Copy + Clone + PartialEq + Eq + Debug`** derives. `Hash` where the wrapped type supports it.

For wrappers around composite values (e.g. `Bits<N, S>` itself wrapping the strategy-dispatched container), the `Deref<Target = ...>` target may need to be the dispatched container associated type rather than a single bare primitive. The point of the rule is that `*wrapper` always reads through one level of wrapping, never N levels of `wrapper.0.0.0`.

### Macro: `meta_bits_wrapper!`

The Deref / AsRef / From / Into / const-fn-helper / `ZERO` / `ONE` block is identical across `IBits`, `FBits`, `Width`. DRY it via a declarative macro in `mock/crates/arvo/src/newtype.rs`:

```rust
/// Generate a meta-bits ConstParamTy newtype wrapping `Bits<$n, Hot>`,
/// with Deref / AsRef / From to/from `u8`, plus a lowercase const-fn
/// helper for const-generic-position construction.
macro_rules! meta_bits_wrapper {
    (
        $(#[$meta:meta])*
        $vis:vis struct $W:ident wraps Bits<$n:literal, Hot>;
        helper $helper:ident;
    ) => {
        $(#[$meta])*
        #[derive(ConstParamTy, PartialEq, Eq, Copy, Clone, Debug)]
        #[repr(transparent)]
        $vis struct $W(pub $crate::bits::Bits<$n, $crate::strategy::Hot>);

        impl $W {
            pub const ZERO: Self =
                Self($crate::bits::Bits::<$n, $crate::strategy::Hot>::from_raw_u8(0));
            pub const ONE: Self =
                Self($crate::bits::Bits::<$n, $crate::strategy::Hot>::from_raw_u8(1));
        }

        impl core::ops::Deref for $W {
            type Target = u8;
            #[inline(always)]
            fn deref(&self) -> &u8 { &*self.0 }
        }

        impl AsRef<u8> for $W {
            #[inline(always)]
            fn as_ref(&self) -> &u8 { &*self.0 }
        }

        impl From<u8> for $W {
            #[inline(always)]
            fn from(n: u8) -> Self { $helper(n) }
        }

        impl From<$W> for u8 {
            #[inline(always)]
            fn from(w: $W) -> u8 { w.0.to_raw_u8() }
        }

        /// Const-fn helper for const-generic-position construction.
        ///
        /// The bare `u8` parameter is workspace-rule exception #4
        /// (helper-fn parameter constructing an arvo type from a literal).
        $vis const fn $helper(n: u8) -> $W {
            $W($crate::bits::Bits::<$n, $crate::strategy::Hot>::from_raw_u8(n))
        }
    };
}
```

Invoked once per meta-bits wrapper:

```rust
meta_bits_wrapper! {
    /// Integer-bit count for fixed-point types.
    pub struct IBits wraps Bits<7, Hot>;
    helper ibits;
}
meta_bits_wrapper! {
    /// Fractional-bit count for fixed-point types.
    pub struct FBits wraps Bits<7, Hot>;
    helper fbits;
}
meta_bits_wrapper! {
    /// Bit-width meta value (1..=128).
    pub struct Width wraps Bits<7, Hot>;
    helper width;
}
```

`USize` / `Bool` / `Cap` stay bespoke (USize: cfg-gated platform-width Bits + unsafe Deref through usize-equivalent container; Bool: `Try` impl + bool-not-Bits; Cap: wraps `USize` not `Bits`). If more bool-shape or platform-shape wrappers appear later, sibling macros can DRY them.

### Explicit per-wrapper implementations this round

The round implements the surface on these wrappers:

- **`IBits(pub Bits<7, Hot>)`**: `Deref<Target = u8>` (routes through `Bits<7, Hot>::to_raw_u8()` via field access; const-fn body is `&self.0.0` once `Bits<N, Hot>` itself exposes `Deref` to its inner native, see below). `From<u8>` / `From<IBits> for u8`. `AsRef<u8>`. `ibits(n: u8) -> IBits` const fn (already present).
- **`FBits(pub Bits<7, Hot>)`**: same shape as `IBits`.
- **`Width(pub Bits<7, Hot>)`**: same shape, plus the dual-path const-fn pair `width_u8(Width) -> u8` (for anonymous-const context) and `width_le_64(Width) -> bool` (for Fnv1a's const-eval guard).
- **`USize(pub Bits<{cfg-platform-width}, Hot>)`**: `Deref<Target = usize>` via documented unsafe transmute (per topic §CR-2). `From<usize>` / `From<USize> for usize`. `AsRef<usize>`.
- **`Bits<const N: u8, S: Strategy>`** itself: `Deref<Target = <S as UContainerFor<N>>::T>` returning `&self.0`. This is the load-bearing one — once `Bits` derefs to its container, `IBits`'s deref to `u8` works through one level of wrapping (`*ibits` produces `u8` because `*ibits` resolves through `IBits`'s `Deref` to `Bits<7, Hot>`, then through `Bits<7, Hot>`'s `Deref` to `u8`). The Rust auto-deref chain handles the rest.
- **`Cap(pub USize)`**: existing field stays. Add `Deref<Target = usize>` (chains through `USize`). Add `From<USize>` / `From<Cap> for USize`.
- **`Bool(pub bool)`**: already has Deref / Try / AsBool / `From<bool>` / `From<Bool> for bool`. Verify the set is complete; add anything missing.

### Call-site impact

Post-implementation, the following all work without manual `.0` chains:

```rust
let i: IBits = 7u8.into();              // or IBits::from(7)
let n: u8 = *i;                         // through Deref chain
let i_const: IBits = ibits(7);          // const-generic position
let w: Width = width(28);
const N_BITS: u8 = width_u8(W64);       // anonymous-const context

// Existing literal-construction call sites mechanically migrate:
//   IBits(7)            →  ibits(7)
//   FBits(0)            →  fbits(0)
//   USize(N as usize)   →  USize::from(N as usize)  // or unchanged
```

The mechanical migration `IBits($n)` → `ibits($n)` (and analogous for FBits) is a `sed` sweep across `mock/crates/arvo*/src/` and `mock/crates/arvo*/tests/`. Estimated touch count from the doc-CL grep: under 700 sites; the sweep is mechanical, not site-by-site judgement. The bare-`IBits(7)` form continues to compile during the transition (the tuple-struct constructor still exists; it just takes a `Bits<7, Hot>` argument now), so call sites that pass an already-wrapped value keep working without edit.

### Anti-pattern: `foo.0.0` chains

Source code touched by this round must NOT contain `.0.0` field access chains on arvo wrappers. Every level of unwrapping goes through `Deref` (`*foo`), `as_ref`, `into`, or a typed accessor (`width_u8`, `to_raw_u8`, `to_raw_u64`). If a site needs raw access through two wraps, it gets a typed accessor on the outer wrapper, not an `.0.0` chain.

This is a SRC-phase enforcement convention, not a lint (yet). A future workspace lint `no-multi-field-wrap-access` could flag it.

```rust
//! Hash algorithm primitives.

/// FNV-1a-64 over a byte slice (free const fn).
///
/// Returns the raw 64-bit state. `Fnv1a<N>::hash_const` masks to N
/// bits via `Bits::<{ width_u8(N) }, Hot>::from_raw_u64`. The
/// `&[u8]` parameter is workspace-rule exception #4 (boundary input
/// from raw bytes). The `u64` return is the algorithm's state-width
/// (workspace-rule exception #4: ergonomic helper-fn returning a
/// bare primitive at the algorithm boundary).
pub const fn fnv1a_64(bytes: &[u8]) -> u64 {
    // lint:allow(no-bare-numeric) lint:allow(arvo-types-only) reason: FNV-1a-64 algorithm boundary; raw byte slice in, raw u64 state out per algorithm contract; tracked: #256
    const FNV_OFFSET_BASIS: u64 = 0xcbf29ce484222325;
    const FNV_PRIME: u64 = 0x100000001b3;
    let mut hash: u64 = FNV_OFFSET_BASIS;
    let mut i = 0;
    while i < bytes.len() {
        hash ^= bytes[i] as u64;
        hash = hash.wrapping_mul(FNV_PRIME);
        i += 1;
    }
    hash
}
```

```rust
//! FNV-1a-64 streaming hasher with N-bit output.

use arvo::{Bits, Hot, Width};
use arvo::strategy::{UContainerFor, width_u8, width_le_64};
use crate::Hasher;
use crate::algo::fnv1a_64;

/// Streaming FNV-1a-64 hasher with N-bit output.
///
/// `N <= 64`. For wider widths see the `Fnv1a128` BACKLOG entry.
pub struct Fnv1a<const N: Width>
where
    Hot: UContainerFor<{ width_u8(N) }>,
    [(); 1 / (width_le_64(N) as usize)]: ,
{
    state: u64,  // lint:allow(no-bare-numeric) lint:allow(arvo-types-only) reason: FNV-1a-64 internal state; algorithm-fixed width per RFC; tracked: #256
}

impl<const N: Width> Fnv1a<N>
where
    Hot: UContainerFor<{ width_u8(N) }>,
    [(); 1 / (width_le_64(N) as usize)]: ,
{
    pub const fn new() -> Self {
        Self { state: 0xcbf29ce484222325 }  // lint:allow(no-bare-numeric) reason: FNV offset basis; algorithm-fixed; tracked: #256
    }

    /// Compile-time hash construction.
    pub const fn hash_const(bytes: &[u8]) -> Bits<{ width_u8(N) }, Hot> {
        let raw = fnv1a_64(bytes);
        Bits::<{ width_u8(N) }, Hot>::from_raw_u64(raw)
    }
}

impl<const N: Width> Hasher<N> for Fnv1a<N>
where
    Hot: UContainerFor<{ width_u8(N) }>,
    [(); 1 / (width_le_64(N) as usize)]: ,
{
    fn update(&mut self, bytes: &[u8]) {
        let mut i = 0;
        while i < bytes.len() {
            self.state ^= bytes[i] as u64;
            self.state = self.state.wrapping_mul(0x100000001b3);
            i += 1;
        }
    }

    fn finalize(self) -> Bits<{ width_u8(N) }, Hot> {
        Bits::<{ width_u8(N) }, Hot>::from_raw_u64(self.state)
    }
}
```
