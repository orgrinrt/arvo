# Src Changelist: arvo L2 Cap/USize cascade cleanup

**Date:** 2026-04-26
**Phase:** SRC
**Scope:** `arvo-bitmask`, `arvo-graph`, `arvo-sparse` crates.
**Doc CL:** 202604260000_changelist.doc.lock.md

## Deliverables

### arvo-bitmask

- `pub const fn cap_size(c: Cap) -> usize { c.0.0 }` at module
  `matrix`, re-exported from the crate root.
- `BitMatrix64<const N: Cap>` (was `const N: usize`). Storage
  `[Mask64; cap_size(N)]` with `where [(); cap_size(N)]:` on the
  struct and every impl. `BitMatrix256<const N: Cap>` the same.
- `propagate_dirty_64<const N: Cap>(...)` and
  `propagate_dirty_256<const N: Cap>(...)` signatures + bodies.
  Internal loop bounds switch from `N` to `cap_size(N)`.
- `matrix.rs` header `//!` comment rewritten: remove the incorrect
  "Const-generic N: usize rather than arvo::Cap" paragraph and
  replace it with a short description of the `const N: Cap` + 
  `cap_size` pattern.
- Tests (`tests/bitmatrix.rs`, `tests/dirty.rs`) updated:
  `#![feature(adt_const_params)]` added; a local `const fn cap(n:
  usize) -> Cap { Cap(USize(n)) }` helper defined; per-size named
  consts (`C4`, `C5`, `C6`, `C8`) used at call sites.

### arvo-graph

- Import `cap_size` from `arvo_bitmask` in every file that takes
  `BitMatrix64<const N: Cap>`.
- Every `fn X<const N: usize>(...)` → `fn X<const N: Cap>(...)`,
  plus `where [(); cap_size(N)]:` where the body touches arrays or
  indexes `N`.
- Array return / storage types `[T; N]` become `[T; cap_size(N)]`
  (private locals may use inline `cap_size(N)`; public signatures
  use the `cap_size(N)` form in the type position).
- `SpanningTree<const N: Cap>` (was `const N: usize`). Internal
  `[NodeId; N]` fields become `[NodeId; cap_size(N)]`.
- Affected files: `src/topo.rs`, `src/rank.rs`, `src/components.rs`,
  `src/path.rs`, `src/waist.rs`, `src/spanning.rs`.
- Crate root `#![feature(adt_const_params)]` added (alongside
  existing `generic_const_exprs`).
- Tests (`tests/topo.rs`, `tests/rank.rs`, `tests/components.rs`,
  `tests/path.rs`, `tests/waist.rs`, `tests/spanning.rs`) updated
  the same way as arvo-bitmask tests — local `cap` helper + named
  consts.

### arvo-sparse

- Import `cap_size` from `arvo_bitmask` where consuming
  `BitMatrix64`.
- `Csr<const ROWS: Cap, const NNZ: Cap, W>` (was `usize` for both).
  Storage `[USize; cap_size(ROWS)]`, `[NodeId; cap_size(NNZ)]`,
  `[W; cap_size(NNZ)]` with `where [(); cap_size(ROWS)]:, [();
  cap_size(NNZ)]:`.

  `rcm_reorder_256<const N: Cap>`, `block_diagonal_64<const N:

  `dulmage_mendelsohn_64<const N: Cap>`,
  `dulmage_mendelsohn_256<const N: Cap>`.
  Array returns become `[T; cap_size(N)]`.
- `DulmageMendelsohn<const N: Cap>` struct the same.
- Affected files: `src/csr.rs`, `src/rcm.rs`, `src/block.rs`,
  `src/dm.rs`.
- Crate root `#![feature(adt_const_params)]` added.
- Tests updated like arvo-bitmask / arvo-graph.

## Toolchain

Inherited nightly. All three crates already enable
`generic_const_exprs` + `allow(incomplete_features)`;
`adt_const_params` is the additional feature required so `Cap`
passes as a const-generic parameter.

## Impl-time decisions (pre-committed)

1. **`cap_size` placement.** In arvo-bitmask at the top of
   `src/matrix.rs`, re-exported from `lib.rs`. arvo-graph and
   arvo-sparse import via `use arvo_bitmask::cap_size;`. No local
   copies. Matches how `Mask64` / `BitMatrix64` are imported.

2. **`Cap` import.** Every affected file adds `Cap` to its
   `arvo::newtype::{...}` import where not already present. Usually
   alongside `USize`.

3. **Where-clauses.** Apply `where [(); cap_size(N)]:` uniformly on
   every `impl<const N: Cap>` block and every `fn` taking `const N:
   Cap` — even where technically redundant. Consistent surface for
   readers; the compiler ignores truly-redundant bounds.

4. **Test helper.** In each test module:
   ```rust
   const fn cap(n: usize) -> Cap { Cap(USize(n)) }
   const C4: Cap = cap(4);
   const C6: Cap = cap(6);
   // ...as needed per file
   ```
   Keeps test bodies readable (`BitMatrix64<C8>` vs the full
   newtype literal).

5. **Internal loop bounds.** Convert `for i in 0..N` → `for i in
   0..cap_size(N)`. Internal `usize` index variables stay raw — the
   public surface already uses `NodeId` or `USize` for indices.

6. **Cascade order within impl.** arvo-bitmask first (breaks
   downstream builds intentionally), then arvo-graph, then
   arvo-sparse. Commit after each crate. The workspace builds again
   once all three are aligned.

7. **Span changes.** No source-level logic changes, no behaviour
   changes. Every existing test passes unchanged (after type
   signature of the test itself is updated).

8. **No `#[inline]` additions or removals.** Helper `cap_size`
   is `const fn`; the inliner handles it trivially.
