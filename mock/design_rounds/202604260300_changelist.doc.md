## Purpose

Final sliver of the arvo L2 Cap/USize cleanup series (Task #51).
arvo-comb does not consume `BitMatrix*`, so it was naturally
independent of the cascade round 202604260000 that aligned
arvo-bitmask / arvo-graph / arvo-sparse. This round applies the
same newtype-on-public-surfaces pattern to arvo-comb's three
public algorithms.

Corrections:

1. **`const N: Cap`, not `const N: usize`.** Every public fn signature
   takes the arvo `Cap` newtype as its const-generic size parameter.
   Multi-parameter fns (`greedy_group<N, M>`, `bin_pack<N, B>`) swap
   each `usize` to `Cap`. Array sizing via
   `cap_size(c: Cap) -> usize { c.0.0 }` imported from arvo-bitmask.

2. **`USize` over `usize`** in public count/index positions.
   Internal `usize` loop locals stay.

3. Private helper `fill_splits<const N: usize>` also cascades to
   `const N: Cap` to stay consistent with the public caller's
   generic parameter.

`cap_size` imports from arvo-bitmask. Same tracking entry as
arvo-graph/arvo-sparse regarding future L0 promotion.

No algorithmic changes. Stack storage layouts unchanged.
