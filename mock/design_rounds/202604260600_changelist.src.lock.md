**Scope:** `arvo` crate only.

## Deliverables

- Add `core::ops::{Add, Sub, Mul, Div, Neg}` impls for
  `FastFloat<F: Ieee>` delegating to the wrapped `F`. `Output =
  Self`. Five impls, one macro invocation.
- Same for `StrictFloat<F: Ieee>`. Five impls.
- The `F: Ieee + core::ops::<Op><Output = F>` bound is sufficient;
  no change to the `Ieee` trait's supertrait list (keeps the seal
  simple, keeps unrelated Ieee consumers compiling unchanged).
- Unit tests covering add, sub, mul, div, neg for both wrappers at
  both widths (f32 + f64). Basic round-trip: `a op b == FF(a.0 op
  b.0)`.

Out of scope:
- `core::ops::{Rem, Shl, Shr, BitAnd, BitOr, BitXor, Not}` — not
  relevant for floats.
- `AddAssign` / `SubAssign` / etc. — not needed by current
  consumers. Add in follow-up round if surfaced.
- Any change to `Ieee` supertrait bounds.
- DESIGN or BACKLOG edits.

## Impl-time decisions (pre-committed)

1. **Delegation shape.** Each op is a one-line delegation:
   ```rust
   impl<F: Ieee + Add<Output = F>> Add for FastFloat<F> {
       type Output = Self;
       #[inline(always)]
       fn add(self, other: Self) -> Self {
           Self(self.0 + other.0)
       }
   }
   ```
   For `Neg` the `Output = Self` still applies; the bound is
   `F: Ieee + Neg<Output = F>`.

2. **Macro to reduce duplication.** Define a `float_op_impl!`
   macro locally in `float.rs`:
   ```rust
   macro_rules! float_op_impl {
       ($wrapper:ident, $op:ident, $method:ident) => {
           impl<F: Ieee + core::ops::$op<Output = F>> core::ops::$op
               for $wrapper<F>
           {
               type Output = Self;
               #[inline(always)]
               fn $method(self, other: Self) -> Self {
                   Self(core::ops::$op::$method(self.0, other.0))
               }
           }
       };
   }
   ```
   Then call it 4 × 2 = 8 times for the binary ops; `Neg` needs a
   separate macro arm because it's unary. Total: 8 binary-op
   invocations + 2 `Neg` impls (hand-written or via a `neg_impl!`
   arm).

3. **No `#[inline(always)]` elsewhere.** The existing float-op
   impls use `#[inline(always)]`; match that convention for
   consistency.

4. **Test file placement.** New integration test file
   `tests/float_ops.rs` rather than module-level unit tests — keeps
   the existing `float.rs` module uncluttered and mirrors how
   `tests/ufixed.rs` / `tests/ifixed.rs` are organised.

5. **Test coverage.** One test per op per wrapper per width = 5 × 2
   × 2 = 20 tests. Use simple, exact-representable values (2.0,
   3.0, etc.) so bit-exact comparisons work on both f32 and f64
   without epsilon fuzz.

6. **No changes to existing float impls.** Sqrt, Recip, TotalOrd,
   Abs, FromConstant stay exactly as they are.
