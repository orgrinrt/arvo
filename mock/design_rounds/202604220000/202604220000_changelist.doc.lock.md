## Purpose

Two small clarifications before arvo-graph impl:

1. **BitMatrix dispatch.** The existing DESIGN uses `BitMatrix<const
   N: Cap>` as shorthand. arvo-bitmask (just closed) ships two
   concrete adjacency-matrix types: `BitMatrix64<const N: usize>`
   and `BitMatrix256<const N: usize>`. This round scopes arvo-graph
   algorithms to `BitMatrix64` only (covers up to 64 nodes — the
   common case per polka-dots research). `BitMatrix256` variants
   are deferred to `BACKLOG.md.tmpl` for a follow-up round. The
   generic signatures in DESIGN that read `BitMatrix<N>` should
   spell `BitMatrix64<N>` concretely.

2. **Const-generic parameter type.** The DESIGN shorthand uses `const
   N: Cap` (arvo's wrapper newtype). arvo-bitmask ships `const N:
   usize`. Keep them aligned: arvo-graph also uses `const N: usize`.

These are naming/dispatch clarifications, not design changes. The
algorithm set (topo sort, renumber, upward/downward rank, connected
components, longest path, waist detection, spanning tree) stays as
designed.
