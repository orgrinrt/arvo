## Purpose

One small design correction before arvo-bits impl:

The arvo-bits DESIGN.md.tmpl (shipped in the L0 foundation round)
described `BitAccess` / `BitSequence` as blanket-impl over a `T:
BitPresentation` bound. The L0 `BitPresentation` marker exposes only
`LOGICAL_WIDTH` and has no path to the raw container, so a blanket
impl cannot compile. Resolving as concrete impls on `UFixed<I, F, S>`
and `IFixed<I, F, S>` (per topic 202604200000). No L0 change; this is
a downstream-impl choice, not a substrate-API change.

## Per-crate doc changes

### arvo-bits — `mock/crates/arvo-bits/DESIGN.md.tmpl`

Replace the "blanket impl on `T: BitPresentation`" language under the
"Type family marker traits" section with "concrete impls on
`UFixed<I, F, S>` and `IFixed<I, F, S>`, mediated by a sealed
`BitPrim` / `IBitPrim` helper trait internal to arvo-bits". Add a
short paragraph noting the orphan-rule rationale and the `to_raw` /
`from_raw` delegation path. Keep the trait surface
(`BitWidth` / `BitAccess` / `BitSequence`) and the alias list
(`Bit` / `Nibble` / `Byte` / `Word` / `DWord` / `QWord`) unchanged.
