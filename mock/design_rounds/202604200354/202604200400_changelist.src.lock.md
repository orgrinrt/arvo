### F1 — wrapping div-by-zero fix (CRITICAL)

**File:** `mock/crates/arvo/src/strategy.rs`

Replace the buggy guards in `impl_u_arith_wrapping!` and
`impl_i_arith_wrapping!` with a simple "return numerator on
divide-by-zero" branch. Option A per the doc CL.

```rust
fn u_div(a: ..., b: ...) -> ... {
    if b == <...T as Default>::default() {
        a  // wrapping has no identity for zero; return numerator.
    } else {
        a.wrapping_div(b)
    }
}
```

Same pattern for `i_div`. The buggy
`a.wrapping_div(zero.wrapping_add(zero))` recursion is removed.

Update the strategy.rs module-level comment (lines 384-387) to
remove the "panics in debug on some platforms" language. New
semantic: wrapping div-by-zero returns the numerator unchanged,
never panics, matching the Hot / Warm single-instruction contract.

### F2 — k_way_partition stack overflow-drop (CRITICAL)

**File:** `mock/crates/arvo-spectral/src/partition.rs`

Replace the two independent `stack_len < k` guards with a unified
"both halves or neither" guard. When only one slot is left, push
the half with more nodes (best-effort degradation). Document the
truncation in the fn doc-comment.

```rust
let pos_size = *positive_half.count();
let neg_size = *negative_half.count();

if stack_len + 2 <= cap_size(K) {
    if pos_size > 1 { stack[stack_len] = positive_half; stack_len += 1; }
    if neg_size > 1 { stack[stack_len] = negative_half; stack_len += 1; }
} else if stack_len + 1 <= cap_size(K) {
    if pos_size > neg_size {
        if pos_size > 1 { stack[stack_len] = positive_half; stack_len += 1; }
    } else {
        if neg_size > 1 { stack[stack_len] = negative_half; stack_len += 1; }
    }
}
```

### F3 — sqrt 7-ULP warning (IMPORTANT)

**File:** `mock/crates/arvo/src/traits.rs`

Expand the existing "Substrate placeholder" comment-block into
explicit doc comments on `sqrt_f32` and `sqrt_f64`:

```text
# WARNING: 7-ULP Newton-Raphson approximation

This is not a correctly-rounded sqrt. Three (f32) / four (f64)
NR iterations land within ~7 ULP; the IEEE 754 bit-exact contract
that StrictFloat promises is weakened to "deterministic but not
correctly-rounded" in no_std builds. Consumer code that needs
correctly-rounded sqrt must link libm at a higher layer until the
substrate BACKLOG item ships.
```

No change to computation. Option B per the doc CL.
