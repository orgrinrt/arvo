## Purpose

Same naming/dispatch tightening arvo-graph got, applied to
arvo-sparse. The shipping adjacency-matrix type from arvo-bitmask is
`BitMatrix64<const N: usize>`; the DESIGN uses `BitMatrix<const N:
Cap>` and `Mask` as shorthand. Align with arvo-bitmask's concrete
types.

Scope: `BitMatrix64` / `Mask64`-backed this round. `BitMatrix256` /
`Mask256` variants of every algorithm deferred to `BACKLOG.md.tmpl`
(same reasoning as arvo-graph).

The `Csr<ROWS, NNZ, W>` structure stays as designed — fixed-size
arrays keyed on `usize` const generics (not `Cap`).

## Per-crate doc changes

### arvo-sparse — `mock/crates/arvo-sparse/DESIGN.md.tmpl`

- Replace `BitMatrix<const N: Cap>` with `BitMatrix64<const N: usize>`.
- Replace bare `Mask` in return/field positions with `Mask64`.
- Replace `const ROWS: Cap` / `const NNZ: Cap` in `Csr<...>` with `const ROWS: usize` / `const NNZ: usize`.
- Add a "Capacity" note: ships `BitMatrix64` / `Mask64`-backed. `BitMatrix256` / `Mask256` variants tracked in `BACKLOG.md.tmpl`.
