## Impl-time decisions (pre-committed)

1. **`greedy_group` overflow handling.** If the algorithm would produce more than `M` groups, it returns early with `group_count == M` and the input items beyond the last group boundary are dropped from the result. The caller sets `M` defensively; the function reports the count, never panics.

2. **`matrix_chain_dp` infeasibility.** If `feasible(i, j)` returns false for all splits of interval `[i..j]`, that interval gets `W::from_constant(u8::MAX)`-equivalent sentinel cost. Simpler: use `TotalOrd::total_cmp` with a "never selected" flag tracked in a parallel `[[bool; N]; N]` feasibility table. Impl picks whichever is cleaner at src time.

3. **`bin_pack` affinity ordering.** Pre-sort item indices by pairwise affinity (high affinity first) using insertion sort on the index array (`[USize; N]`), then first-fit into bins. No external sort; inline swap-based sort — const-time for bounded N.

4. **No `#[inline]` this round.**

5. **Feature flags crate-root**: `#![no_std]`, `#![feature(adt_const_params, generic_const_exprs)]`, `#![allow(incomplete_features)]`.
