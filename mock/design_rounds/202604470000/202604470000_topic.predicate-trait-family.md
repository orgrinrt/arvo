# Current state

Two consumer sites in arvo-comb take predicate-shaped callbacks:

| Site | Signature |
|------|-----------|
| `arvo-comb/src/dp.rs:37` | `feasible: impl Fn(USize, USize) -> Bool` |
| `arvo-comb/src/greedy.rs:36` | `feasible: impl Fn(&A, &T) -> Bool` |

These work, but the bound is unnamed: it's just "anything callable that
returns Bool". Algorithmic crates carry these predicates through
internal machinery; the bound would read better as a named trait and
hook consumers implementing their own callable types (say, a state
machine with a `test` method, not a closure).

No Predicate trait exists today. Raw `impl Fn(...) -> Bool` is the
pattern.

# Changes proposed

Ship a three-arity predicate trait family in `arvo`:

```rust
pub trait Pred<A> {
    fn test(&self, a: &A) -> Bool;
}
pub trait Pred2<A, B> {
    fn test(&self, a: &A, b: &B) -> Bool;
}
pub trait Pred3<A, B, C> {
    fn test(&self, a: &A, b: &B, c: &C) -> Bool;
}
```

Blanket Fn-bridge impls make any closure a Pred automatically:

```rust
impl<A, F: Fn(&A) -> Bool> Pred<A> for F { ... }
impl<A, B, F: Fn(&A, &B) -> Bool> Pred2<A, B> for F { ... }
impl<A, B, C, F: Fn(&A, &B, &C) -> Bool> Pred3<A, B, C> for F { ... }
```

Three arities cover the shipped sites and the closest parallel arity
to `FromConstant`-family traits. Additional arities can be added in a
follow-up round.

New file: `mock/crates/arvo/src/predicate.rs`.

## Why a value-taking method vs by-reference Fn?

The `Fn(A) -> Bool` sites in dp.rs pass by value (USize, USize), not
by reference. The new trait uses `&A` uniformly:

- matches Rust's standard predicate shape (`Iterator::filter`, etc.)
- plays nicely with non-Copy keys/items in greedy.rs
- copy-able types (USize, u*) lose nothing at call time; the compiler
  optimises the reference away

The dp.rs site switches from `Fn(USize, USize) -> Bool` to
`Pred2<USize, USize>` and dereferences once at the test call.

# Non-goals

- No 0-ary predicate (niladic callable is just a function pointer;
  no semantic gain).
- No 4+ arity (not needed by any shipped site).
- No async / fallible variants; Pred returns plain Bool.
- No `dyn` support; the family is monomorphised just like everything
  else in arvo.
