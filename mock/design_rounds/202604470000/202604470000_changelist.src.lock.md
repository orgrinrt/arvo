Three named predicate traits with blanket Fn-impl bridges:

```rust
//! Named predicate bounds for consumer callbacks.
//!
//! `Pred<A>`, `Pred2<A, B>`, `Pred3<A, B, C>` name the shape
//! "callable returning Bool" at 1, 2, and 3 arities. Blanket
//! implementations bridge any matching `Fn` closure, so consumers
//! keep passing `|a| ...` lambdas; custom callable types implement
//! `test` directly for stateful predicates.

use crate::newtype::Bool;

/// Unary predicate.
pub trait Pred<A> {
    fn test(&self, a: &A) -> Bool;
}
impl<A, F: Fn(&A) -> Bool> Pred<A> for F {
    #[inline(always)]
    fn test(&self, a: &A) -> Bool { self(a) }
}

/// Binary predicate.
pub trait Pred2<A, B> {
    fn test(&self, a: &A, b: &B) -> Bool;
}
impl<A, B, F: Fn(&A, &B) -> Bool> Pred2<A, B> for F {
    #[inline(always)]
    fn test(&self, a: &A, b: &B) -> Bool { self(a, b) }
}

/// Ternary predicate.
pub trait Pred3<A, B, C> {
    fn test(&self, a: &A, b: &B, c: &C) -> Bool;
}
impl<A, B, C, F: Fn(&A, &B, &C) -> Bool> Pred3<A, B, C> for F {
    #[inline(always)]
    fn test(&self, a: &A, b: &B, c: &C) -> Bool { self(a, b, c) }
}
```
