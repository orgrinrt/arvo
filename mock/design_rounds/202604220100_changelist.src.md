## Deliverables

Rust source under `mock/crates/arvo-graph/src/` implementing:

- `topo_sort<const N: usize>(dag: &BitMatrix64<N>) -> (USize, [NodeId; N])` — Kahn's algorithm. Returns `(valid_count, topo_order)`. `valid_count < N` indicates a cycle.
- `renumber<const N: usize>(topo_order: &[NodeId; N]) -> [NodeId; N]` — inverse permutation mapping old → new ID given topo order.
- `upward_rank<const N: usize, W>(dag: &BitMatrix64<N>, weights: &[W; N]) -> [W; N]` with `W: Add<Output = W> + TotalOrd + Copy + FromConstant`.
- `downward_rank<const N: usize, W>(...) -> [W; N]` — same bound, reverse direction.
- `components<const N: usize>(dag: &BitMatrix64<N>) -> [USize; N]` — DFS via `Mask64` visited-set + `[NodeId; N]` stack.
- `longest_path<const N: usize, W>(dag: &BitMatrix64<N>, weights: &[W; N], topo_order: &[NodeId; N]) -> (W, Mask64, [NodeId; N])`.
- `waist_detect<const N: usize>(dag: &BitMatrix64<N>, topo_order: &[NodeId; N]) -> Mask64`.
- `SpanningTree<const N: usize> { on_trunk: Mask64, trunk_next: [NodeId; N], branch_roots: Mask64, bridges: Mask64 }` + `spanning_tree<N, W>(dag: &BitMatrix64<N>, ranks: &[W; N]) -> SpanningTree<N>` with `W: TotalOrd + Copy`.
- Unit tests: known-shape DAGs with expected topo / rank / component / longest-path / waist / spanning-tree outputs.

## Impl-time decisions (pre-committed)

1. **Data structures.** Fixed `[NodeId; N]` queues and stacks. `Mask64` for visited sets and result masks. `[USize; N]` for in-degree and component-ID arrays. All arrays are stack-allocated, const-sized. Zero heap.

2. **`topo_sort` cycle handling.** Kahn's algorithm: start with in-degree-0 nodes. As each is popped, decrement successor in-degrees. Cycle detected when no more zero-in-degree nodes remain but not all have been sorted. `valid_count` records how many were popped before stalling.

3. **`upward_rank` / `downward_rank`.** Two-pass over nodes in reverse topo order (upward) / forward topo order (downward). Rank = own-weight + max(successor-ranks) or + max(predecessor-ranks). `TotalOrd::total_cmp` used for max selection. `FromConstant::from_constant(0)` for leaf / root init.

4. **`components`.** Classic DFS. For each unvisited node, DFS its reachable set, all reached nodes get the next fresh component-ID. `Mask64` visited-set + `[NodeId; N]` stack.

5. **`longest_path`.** One-pass DP in topo order. For each node `v`, iterate predecessors (scan all rows for bit `v` set), compute `rank[p] + weight[v]`, keep the max. Track the best predecessor for each node in `[NodeId; N]` predecessor array. Roots (no predecessors) get `Mask64` bit unset in `has_predecessor`.

6. **`waist_detect`.** For each topo level (group of nodes with the same topo depth), count nodes. Local minima = waist points. Topo depth computed as max predecessor depth + 1. Emit a `Mask64` with bits set at waist positions in the TOPO ORDER (not the node IDs).

7. **`spanning_tree`.** Start with highest-ranked root. Descend to highest-ranked successor repeatedly — that's the trunk. At each fan-out, non-trunk successors become branches (recursively decomposed). At each fan-in, the receiving node is a bridge.

8. **No `#[inline]` annotations for this round.** Compiler inlining is fine; we're not benchmarking. Hot-path annotation is a benchmark-round concern.
