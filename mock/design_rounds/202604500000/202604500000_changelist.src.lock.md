## `mock/benches/src/main.rs` (create)

Bench bin source. Carries:

1. `Fnv1aVsXxHash3<const N: usize>` Routine with `Input = [u8; N]`,
   `Output = u64`, `outputs_may_differ() -> true`, and a SplitMix64-
   style seed-driven `build_input`.
2. `routine_bridge!` macro invocations for each declared size, used
   only by the worker re-exec path (the orchestrator uses harness
   API directly).
3. `main()` dispatching to `run_orchestrator()` or `run_worker()`
   based on `--worker` arg.
4. `run_orchestrator()` reading bench.toml, resolving variant paths
   via `shape_variant_path`, invoking `harness::run`.
5. `run_worker()` reading the worker payload from stdin and writing
   sample output to stdout.
6. `shape_variant_path()` mapping a bare cargo target stem like
   `../target/release/fnv1a` to `../target/release/libfnv1a.dylib`
   (or `.so` / `.dll`) via `std::env::consts::DLL_PREFIX` /
   `DLL_SUFFIX`.

The `current_dir` lookup uses `.expect(...)` per the mockspace
self-bench's reviewer-fix posture (no silent fallback to `.`).

## `mock/benches/variants/fnv1a/src/lib.rs` (create)

Wraps `arvo_hash::Fnv1a` and writes the digest into the output u64
via `#[bench_variant(...)]`:

```rust
//! FNV1a variant for the ContentHash bench.

use arvo_hash::{Fnv1a, HasherExt};
use mockspace_bench_macro::bench_variant;

mod bench {
    use super::*;
    pub struct Fnv1aVsXxHash3<const N: usize>;
    impl<const N: usize> mockspace_bench_core::Routine
        for Fnv1aVsXxHash3<N>
    {
        type Input = [u8; N];
        type Output = u64;
        fn build_input(_seed: u64) -> [u8; N] { [0u8; N] }
        fn ops_per_call(_input: &[u8; N]) -> u64 { N as u64 }
        fn outputs_may_differ() -> bool { true }
    }
}

use bench::Fnv1aVsXxHash3;

#[bench_variant(Fnv1aVsXxHash3, "fnv1a", sizes = [64, 256, 1024, 4096])]
fn run_fnv1a<const N: usize>(input: &[u8; N], output: &mut u64) {
    let h = Fnv1a::<64>::new().hash(input);
    *output = h.into();
}
```

The Routine impl is duplicated in both variant cdylibs because each
cdylib needs the type to satisfy the bench_variant macro's bound
`Algo<N>: Routine`. The duplicate is mechanical and small. (A v3
follow-up under #281 may hoist a shared type into a sibling crate
once the harness offers a clean way; today, duplication is the
correct cost.)
