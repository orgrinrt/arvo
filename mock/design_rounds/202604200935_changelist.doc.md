## Purpose

Land the arvo side of the three-repo primitive-vocabulary retrofit
(Task #71). Replace bare `Option<T>` / `Result<T, E>` on own-trait
method signatures and own inherent methods with `notko::Maybe<T>` /
`notko::Outcome<T, E>`. Keep std-trait impls (`TryFrom`, `Iterator`,
`PartialOrd`) on `Result`/`Option` — those are the designated
escape-hatch. Wrap drift that cannot be removed (const-generic
accessors returning `u8`/`usize`) with tracked allow comments + a
BACKLOG note for future removal.

arvo-bits / arvo-graph / arvo-comb / arvo-spectral have no Option or
Result on public surfaces today; they remain untouched. arvo-bitmask's
only drift is `Vec<_>` in two tests; those switch to fixed-size
arrays.

## TryFrom boundary pattern (canonical)

Re-used across all four sites in `ufixed.rs` (2) and `ifixed.rs` (2):

```rust
impl<const I: IBits, const F: FBits> TryFrom<UFixed<I, F, Warm>>
    for UFixed<I, F, Hot>
where
    Warm: UContainerFor<{ ufixed_bits(I, F) }>,
    Hot: UNarrowFrom<Warm, { ufixed_bits(I, F) }>,
{
    type Error = ();
    #[inline(always)]
    fn try_from(src: UFixed<I, F, Warm>) -> Result<Self, Self::Error> {
        match <Hot as UNarrowFrom<Warm, { ufixed_bits(I, F) }>>::u_try_narrow(src.to_raw()) {
            Outcome::Ok(v) => Ok(Self::from_raw(v)),
            Outcome::Err(()) => Err(()),
        }
    }
}
```

The `Outcome` import lands alongside the existing `use crate::strategy::*;`
line in both files. No `From<Outcome> for Result` conversion is
added to notko; the match is the adapter.
