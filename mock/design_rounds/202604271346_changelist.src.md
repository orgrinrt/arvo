It enumerates every `mock/Cargo.toml`, `mock/crates/*/Cargo.toml`, `mock/crates/*/src/**/*.rs`, `mock/crates/*/build.rs`, `mock/crates/*/tests/**/*.rs`, and `mock/mockspace.toml` edit required to reach the target architecture. Each pass leaves the workspace in a `cargo check` clean state where possible; the few transitional states get explicit notes.

The nine passes:

1. Crate creation pass (eight new crate skeletons).
2. L0 migration pass (move `Bits`, strategy markers, `Bool`, `USize`, typed meta-bits to `arvo-storage`).
3. Contract crate population pass (split trait declarations into the six contract crates).
4. Facade reshape pass (arvo's `src/lib.rs` becomes re-exports + facade-specific types).
5. Per-size helper deletion pass (remove `from_raw_uN` / `to_raw_uN` / `From<uN>` / `impl_bits_uN` macro family).
6. Bitfield macro rewrite pass (emitted-body builder uses const-trait method delegation).
7. Q-D alias deletion pass (drop every non-canonical `Uint*` / `Int*` alias).
8. `mockspace.toml` update pass (forbidden-imports scopes for the new crates).
9. Validation pass (`cargo check`, `cargo mock --lint-only --commit/--push/--build`, full test suite).

fn field(self) -> Uint<N> {
