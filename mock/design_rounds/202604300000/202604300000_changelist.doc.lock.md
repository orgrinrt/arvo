# Doc Changelist: Fix Bits<N> container dispatch; add Bitfield<N>

**Date:** 2026-04-21
**Phase:** DOC
**Topic:** `202604300000_topic.arvo-bits-fix-and-bitfield.md`

## Design pivot (recorded mid-round)

The topic proposed `Bits<N>` with its own `BitContainerFor<N>`
trait. Mid-round discussion converged on a simpler shape: make
`Bits<N>` a `#[repr(transparent)]` newtype around `UFixed<{IBits(N)},
{FBits::ZERO}, S>`, reusing UFixed's existing `UContainerFor<S, N>`
machinery. Named aliases (`Bit` / `Nibble` / `Byte` / `Word` /
`DWord` / `QWord`) collapse onto `Bits<N>` — one primitive family.
Arithmetic is deliberately absent on `Bits` (newtype doesn't inherit
UFixed's `Add` / `Sub` / `Mul` / `Div` impls); bit logic, bit scan,
bit access, bit width all delegate to the inner UFixed.

Bitfield accessors return `Bits<W, Hot>` for a W-bit sub-range —
the exact-width arvo primitive, never a raw u8/u16/u32.

## `mock/crates/arvo-bits/DESIGN.md.tmpl`

### "Opaque-bit container: Bits"

Rewrite to the newtype-over-UFixed definition:

- `Bits<const N: u8, S: Strategy = Hot>` is `#[repr(transparent)]`
  over `UFixed<{IBits(N)}, {FBits::ZERO}, S>`.
- Container dispatch rides on UFixed's existing `UContainerFor<S, N>`
  (u8/u16/u32/u64 per N under Hot).
- `size_of::<Bits<N, Hot>>()` matches the container: 1/2/4/8 bytes.
- No `BitContainerFor<N>` trait in arvo-bits — reuse UFixed's.
- Newtype on purpose: type aliases inherit every trait impl, so the
  opaque-identity semantic requires a newtype to opt out of arithmetic.
- `Bits::from_raw` / `to_raw` delegate to UFixed; `Bits::new(u64)` /
  `bits(self) -> u64` are per-N const-fn bridges for legacy consumers
  that already think in u64 (arvo-hash digests, hilavitkutin-str
  handles).
- Delegates `BitWidth` / `BitAccess` / `BitSequence` / `BitLogic` to
  the inner UFixed; no `Add` / `Sub` / `Mul` / `Div` / `Ord`.

### "Semantic aliases"

Redefine as `Bits<N>` aliases (not parallel UFixed aliases):

```rust
pub type Bit<S: Strategy = Hot>    = Bits<1,  S>;
pub type Nibble<S: Strategy = Hot> = Bits<4,  S>;
pub type Byte<S: Strategy = Hot>   = Bits<8,  S>;
pub type Word<S: Strategy = Hot>   = Bits<16, S>;
pub type DWord<S: Strategy = Hot>  = Bits<32, S>;
pub type QWord<S: Strategy = Hot>  = Bits<64, S>;
```

One family. The aliases shed arithmetic along with `Bits`.
Consumers that want arithmetic reach for `UFixed<{IBits(N)},
{FBits::ZERO}, S>` directly.

### New section "Bitfield — packed layouts with named sub-ranges"

Documents the `bitfield!` macro. Accessor return types are
`Bits<W, Hot>` per sub-range width `W` (or the named alias for
common `W`). Setters take the same type. Compile-time checks:
disjoint sub-ranges, all fit within N, N ≤ 64.
