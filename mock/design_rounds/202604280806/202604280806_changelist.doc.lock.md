**Goal:** Document the new axis traits that decompose Strategy into orthogonal sub-axes.

Mutations:

- **New `## Strategy axis decomposition` section** (insert after `## Strategy markers`): Document the three internal axis traits:
  - `OverflowPolicy`: the strategy's overflow handling (Wrapping / Saturating).
  - `ContainerWidth`: the strategy's container width relative to logical width (Min / DoubleLogical).
  - `StorageLayout`: the strategy's storage layout (Dense / Bitpacked).
  Note that the four existing strategy markers (`Hot`, `Warm`, `Cold`, `Precise`) implement specific axis combinations:
  | Marker | OverflowPolicy | ContainerWidth | StorageLayout |
  |---|---|---|---|
  | Hot | Wrapping | Min | Dense |
  | Warm | Wrapping | DoubleLogical | Dense |
  | Cold | Wrapping | Min | Bitpacked |
  | Precise | Saturating | DoubleLogical | Dense |
  State that the axis traits are documentation + future-proofing for this round; arithmetic / widen / narrow impls do NOT refactor to dispatch via axes in Round C (deferred to a future round).
  Note that the public surface (the four markers) is unchanged.
