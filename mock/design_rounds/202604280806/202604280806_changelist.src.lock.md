## Per-pass execution

Two passes. Pass 1 implements the meta-newtype carrier widening via the `MetaCarrier` parallel-carrier workaround. Pass 2 ships the `OverflowPolicy` / `ContainerWidth` / `StorageLayout` axis traits in arvo-strategy (axis traits only; no arithmetic dispatch refactor).

### Pass 1: MetaCarrier parallel-carrier workaround

`arvo-storage/src/meta_bits.rs`:
- Add `pub struct MetaCarrier(pub u16)` with `repr(transparent)`, derives `ConstParamTy + PartialEq + Eq + Copy + Clone + Debug + Default + Hash`. Document the layout-equivalence invariant against `Bits<9, Hot, Unsigned>` and the rustc cycle-workaround rationale.
- Add inherent methods on MetaCarrier: `from_raw(n: u16) -> Self`, `to_raw(self) -> u16`, `as_bits(self) -> Bits<9, Hot, Unsigned>` (zero-cost transmute via `core::mem::transmute`).
- Update `meta_bits_wrapper!` macro: inner field from `pub u8` to `pub MetaCarrier`. ZERO/ONE constants construct via `MetaCarrier::from_raw`. `raw(self) -> u16` returns `self.0.to_raw()`. `as_bits(self) -> Bits<9, Hot, Unsigned>` accessor forwards through `MetaCarrier::as_bits`.
- Update `Transparent` impl: `type Inner = u16` (transitive transparency through MetaCarrier through u16).
- Update `Deref<Target = u16>` and `AsRef<u16>`: forward through `self.0.to_raw()` (returns u16 by value) — wait, Deref needs `&u16`. Use `&self.0.0` (direct field access on MetaCarrier; we're in arvo-storage so visibility is fine).
- Update `From<u8>` / `From<u16>` impls to match the helper's u16 signature. Add `From<W> for u16` for the carrier width view.
- Update `From<W> for u8` to narrow via `(self.0.to_raw() as u8)`.
- Update helper signature: `$helper(n: u16) -> $W` constructs via `$W(MetaCarrier::from_raw(n))`.
- Update `Width::W64` const to use `width(64u16)`.

`arvo/src/strategy.rs`:
- Update `ufixed_bits(i: IBits, f: FBits) -> u8`: `i.raw()` and `f.raw()` now return u16; cast result to u8.
- Same for `ifixed_bits`.
- `is_fractional(f) -> usize`: `f.raw() == 0` works for u16; no change.
- `width_le_64(n: arvo_storage::Width) -> bool`: `n.raw() <= 64` works for u16; no change.

`arvo-storage/src/lib.rs`:
- Re-export `MetaCarrier` for the documented public surface.

Sweep all `.0` field accesses on `IBits` / `FBits` / `Width` across the workspace: change to `.raw()` (returns u16) or `.as_bits()` as appropriate. Test files using `IBits(N)` / `FBits(N)` / `Width(N)` tuple-struct construction need to migrate to `ibits(N)` / `fbits(N)` / `width(N)` helper calls.

### Pass 2: Axis traits in arvo-strategy

`arvo-strategy/src/axes.rs` (new file):
- Declare three sealed marker traits: `OverflowPolicy`, `ContainerWidth`, `StorageLayout`.
- Six ZST sub-markers: `Wrapping` / `Saturating` (impl OverflowPolicy), `Min` / `DoubleLogical` (impl ContainerWidth), `Dense` / `Bitpacked` (impl StorageLayout).
- Each sub-marker: `repr(transparent)` not strictly required (ZSTs); standard derives + `ConstParamTy` for future use; sealed via existing `sealed::Sealed`.

`arvo-strategy/src/lib.rs`:
- Add `mod axes;` and `pub use axes::{OverflowPolicy, ContainerWidth, StorageLayout, Wrapping, Saturating, Min, DoubleLogical, Dense, Bitpacked};`.
- Add `impl OverflowPolicy + ContainerWidth + StorageLayout` axis-implementation impls on the four strategy markers (Hot / Warm / Cold / Precise) per the doc CL table. Pattern:
  ```rust
  // Strategy = (OverflowPolicy, ContainerWidth, StorageLayout)
  // Hot     = (Wrapping,        Min,            Dense)
  // Warm    = (Wrapping,        DoubleLogical,  Dense)
  // Cold    = (Wrapping,        Min,            Bitpacked)
  // Precise = (Saturating,      DoubleLogical,  Dense)
  ```

  Implementation via marker-as-associated-type trait shape:
  ```rust
  pub trait HasAxes {
      type Overflow: OverflowPolicy;
      type Width: ContainerWidth;
      type Layout: StorageLayout;
  }
  impl HasAxes for Hot {
      type Overflow = Wrapping;
      type Width = Min;
      type Layout = Dense;
  }
  // ... etc.
  ```

  Or, simpler: each axis trait has a marker-bound method or const that returns the chosen variant. The simplest expressible shape is the `HasAxes` trait above. Pick the shape that compiles cleanly under nightly.

The arithmetic / widen / narrow impls do NOT refactor in this round. They continue dispatching via the bundled `Strategy` marker. The axis traits ship as documentation + future-proofing primitives.
