# Src Changelist: arvo-comb Cap/USize cleanup

**Date:** 2026-04-26
**Phase:** SRC
**Scope:** `arvo-comb` crate only.
**Doc CL:** 202604260300_changelist.doc.lock.md

## Deliverables

- `matrix_chain_dp<const N: Cap, W>` (was `const N: usize`). Storage
  `[[W; cap_size(N)]; cap_size(N)]` + `[USize; cap_size(N)]` splits
  array. `where [(); cap_size(N)]:` on the fn.
- Private `fill_splits<const N: Cap>(...)` cascades the same.
- `greedy_group<const N: Cap, const M: Cap, A, T>` (was both
  `usize`). Returns `(USize, [Range; cap_size(M)])`. Input
  `&[T; cap_size(N)]`. `where [(); cap_size(N)]:, [(); cap_size(M)]:`.
- `bin_pack<const N: Cap, const B: Cap, T, W>` (was both `usize`).
  Input `&[T; cap_size(N)]`. Returns `(USize, [USize; cap_size(N)])`.
  `where [(); cap_size(N)]:, [(); cap_size(B)]:` — `B` bound needed
  for internal per-bin accumulator arrays.
- Import `cap_size` from `arvo_bitmask` in each file that needs it
  (dp.rs, greedy.rs, binpack.rs).
- Crate-root `#![feature(adt_const_params)]` added alongside
  existing `generic_const_exprs`.
- Tests updated: each test file gets a local `const fn cap(n: usize)
  -> Cap { Cap(USize(n)) }` + named `C_N` constants at call sites.
  `#![feature(adt_const_params)]` on the test harness.

Out of scope (per doc CL):
- Any algorithm changes.
- `Mask256` bin-pack variant (BACKLOG).
- `cap_size` promotion to arvo L0 (BACKLOG).

## Impl-time decisions (pre-committed)

1. **`cap_size` imported.** `use arvo_bitmask::cap_size;` added to
   each file that uses `const _: Cap` in a signature. No local
   redefinition.

2. **`Cap` import.** `use arvo::newtype::{Cap, USize};` — add `Cap`
   alongside the existing `USize`. The `Range` struct uses `USize`
   already; no change to `Range` itself.

3. **Where-clauses.** `where [(); cap_size(N)]:, [(); cap_size(M)]:`
   (or equivalent with B) on every pub fn that takes multi
   const-generic params. Applied uniformly.

4. **Internal loop bounds.** `for i in 0..N` → `for i in
   0..cap_size(N)`. Internal `usize` index locals stay. Array
   accesses use `cap_size(...)` rather than bare `N`.

5. **Test scope.** Same `const fn cap(n: usize) -> Cap { Cap(USize(n)) }`
   helper as used in arvo-bitmask / arvo-graph / arvo-sparse. Named
   consts at test-file scope (`C4`, `C6`, `C8`, etc. as needed).

6. **No `#[inline]` changes.** Helper call-sites inline trivially.
