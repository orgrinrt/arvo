# Source Changelist: arvo sweep — primitive vocabulary fixup

**Date:** 2026-04-20
**Phase:** SRC
**Scope:** `arvo` + `arvo-sparse` source; `arvo-bitmask` tests only.
**Doc CL:** 202604200935_changelist.doc.lock.md
**Source topics:**
- 202604200930_topic.arvo-sweep-primitive-vocabulary.md

## Purpose

Execute the doc CL: flip arvo's own-trait method signatures from
bare `Result<T, E>` / `Option<T>` to `notko::Outcome<T, E>` /
`notko::Maybe<T>`, adapt the `TryFrom` std-trait impls at the
boundary via match, annotate the three const-generic accessors
that must keep raw-numeric returns, and remove `Vec<_>` collectors
from two arvo-bitmask tests in favour of fixed-size arrays.

## File-by-file changes

### `mock/crates/arvo/src/strategy.rs`

- `use notko::Outcome;` added near the existing `use crate::newtype::{FBits, IBits};` line.
- `pub trait UNarrowFrom<...>` — `fn u_try_narrow(v: Src::T) -> Result<Self::T, ()>` becomes `-> Outcome<Self::T, ()>`. Doc comment unchanged in substance.
- `pub trait INarrowFrom<...>` — matching flip on `i_try_narrow`.
- `macro_rules! impl_u_narrow` — body changes from `if (v as u128) > max_u128 { Err(()) } else { Ok(v as $dst_ty) }` to `if (v as u128) > max_u128 { Outcome::Err(()) } else { Outcome::Ok(v as $dst_ty) }`. Return-type line in the emitted `fn u_try_narrow` header becomes `-> Outcome<$dst_ty, ()>`.
- `macro_rules! impl_i_narrow` — same treatment.
- `pub const fn ibits_u8(i: IBits) -> u8` — prepend `// allow-bare-numeric: tracked: #71` comment line directly above the `#[inline(always)]` attribute.
- `pub const fn fbits_u8(f: FBits) -> u8` — same allow comment.
- `pub const fn is_fractional(f: FBits) -> usize` — same allow comment.

### `mock/crates/arvo/src/ufixed.rs`

Two `impl TryFrom<...>` blocks. Each body flips from:

```rust
<Hot as UNarrowFrom<Warm, { ufixed_bits(I, F) }>>::u_try_narrow(src.to_raw()).map(Self::from_raw)
```

to the boundary-match pattern:

```rust
match <Hot as UNarrowFrom<Warm, { ufixed_bits(I, F) }>>::u_try_narrow(src.to_raw()) {
    Outcome::Ok(v) => Ok(Self::from_raw(v)),
    Outcome::Err(()) => Err(()),
}
```

`use notko::Outcome;` added at the file's top with the existing use-lines.

### `mock/crates/arvo/src/ifixed.rs`

Identical pattern on the two `impl TryFrom<...>` blocks using `INarrowFrom::i_try_narrow`. `use notko::Outcome;` added.

### `mock/crates/arvo-sparse/src/csr.rs`

- `use notko::Maybe;` added at the top alongside existing use-lines.
- `pub fn get(&self, row: USize, col: NodeId) -> Option<W>` — signature flips to `-> Maybe<W>`. Body's two `return None;` become `return Maybe::Isnt;`; terminal `None` becomes `Maybe::Isnt`; `Some(...)` (if any) becomes `Maybe::Is(...)`.

### `mock/crates/arvo-sparse/src/rcm.rs`

- `use notko::Maybe;` added.
- `fn min_degree_unvisited<...>(...) -> Option<usize>` becomes `-> Maybe<USize>`. Local `let mut best: Option<(usize, usize)> = None;` becomes `let mut best: Maybe<(USize, usize)> = Maybe::Isnt;`. `match best` arms use `Maybe::Is` / `Maybe::Isnt`. Terminal `best.map(|(i, _)| i)` becomes `match best { Maybe::Is((i, _)) => Maybe::Is(USize(i)), Maybe::Isnt => Maybe::Isnt }`.
- Any caller in `rcm.rs` that matched on `Some((i, _))` / `None` against the old return type adapts to `Maybe::Is(...)` / `Maybe::Isnt` accordingly. Inner tuple carries `USize` for the node index (external-facing) and a raw `usize` for the internal degree count (private-only, no surface impact).

### `mock/crates/arvo-bitmask/tests/mask_scan.rs`

Two tests (`iter_set_bits_lowest_first`, `iter_set_bits_dense`) collect into `Vec<_>`. Replace with stack arrays:

```rust
let mut collected = [0usize; 64];
let mut n = 0usize;
for u in m.iter_set_bits() {
    collected[n] = u.0;
    n += 1;
}
let got = &collected[..n];
// assertions use `got` in place of the old `collected` Vec slice.
```

Dense test's length assertion (`collected.len() == 64`) becomes `n == 64`; per-element loop over `got.iter().enumerate()`.

### `mock/crates/arvo-bitmask/tests/mask256.rs`

The single `iter_set_bits` collection site gets the same stack-array treatment sized to 256.

### `mock/crates/arvo/tests/cross_width.rs`

No changes to existing `.try_into()` tests — those exercise the std `TryFrom` boundary correctly. Add one small `fn` per u/i pair near the bottom of the file that calls `UNarrowFrom::u_try_narrow` / `INarrowFrom::i_try_narrow` directly and asserts on the returned `Outcome`. Example:

```rust
#[test]
fn u_try_narrow_outcome_ok() {
    use notko::Outcome;
    use arvo::strategy::{Hot, UNarrowFrom, Warm};
    let raw: u16 = 200;
    let got = <Hot as UNarrowFrom<Warm, 8>>::u_try_narrow(raw);
    assert!(matches!(got, Outcome::Ok(200u8)));
}

#[test]
fn u_try_narrow_outcome_err() {
    use notko::Outcome;
    use arvo::strategy::{Hot, UNarrowFrom, Warm};
    let raw: u16 = 300;
    let got = <Hot as UNarrowFrom<Warm, 8>>::u_try_narrow(raw);
    assert!(matches!(got, Outcome::Err(())));
}
```

Parallel pair for `INarrowFrom::i_try_narrow`. Four new tests total.
