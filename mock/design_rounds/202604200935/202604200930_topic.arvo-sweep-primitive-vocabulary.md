## What this round is

The 2026-04-20 fidelity audit surfaced drift between arvo's shipped
types and the workspace-wide primitive-vocabulary discipline (see
`feedback_primitive_vocabulary_discipline.md`). notko now provides
`Maybe<T>` + `Outcome<T, E>` as the canonical replacements for bare
`Option<T>` / `Result<T, E>` in public APIs. arvo uses `USize` for
counts / indices already, so the arvo-specific drift is smaller than
expected — mostly trait methods on arvo's own traits and one
algorithm crate helper.

Scope is the arvo repo only. hilavitkutin + clause consume arvo, but
their sweeps (#72 / #73) are separate rounds.

## In-scope drift (must change)

### Own-trait methods returning bare `Result<T, E>`

- `arvo::strategy::UNarrowFrom::u_try_narrow(v: Src::T) -> Result<Self::T, ()>`
  → `-> Outcome<Self::T, ()>`.
- `arvo::strategy::INarrowFrom::i_try_narrow(v: Src::T) -> Result<Self::T, ()>`
  → `-> Outcome<Self::T, ()>`.
- Macro-emitted `impl UNarrowFrom<_, _> for _` / `impl INarrowFrom<_, _>
  for _` bodies in `strategy.rs` return `Outcome::Ok` / `Outcome::Err`.

### Own inherent method returning bare `Option<T>`

- `arvo_sparse::csr::CsrMatrix::get(&self, row: USize, col: NodeId)
  -> Option<W>` → `-> Maybe<W>`.

### Private helpers inside arvo-sparse that still use `Option`/raw `usize`

- `arvo_sparse::rcm::min_degree_unvisited` — private helper.
  Signature `fn min_degree_unvisited(...) -> Option<usize>` →
  `-> Maybe<USize>`. Local `best: Option<(usize, usize)>` binding →
  `Maybe<(USize, USize)>`. (These are private but the rule applies
  uniformly in arvo; see the type-surface rule.)

### Vec-in-tests

- `arvo-bitmask/tests/{mask_scan,mask256}.rs` collect `iter_set_bits()`
  into `Vec<_>`. Replace with a fixed-size `[USize; N]` + cursor
  pattern. These tests know their expected output length at compile
  time, so bounded arrays are a direct fit.

### Tests that use std `try_into()` → `Result`

- `arvo/tests/cross_width.rs` has `let narrowed: Result<UHot, ()> =
  a.try_into();` style annotations. `.try_into()` routes through
  std `TryFrom` which must return `Result` (std boundary — allowed).
  Keep the `.try_into()` call sites as they are (they test the std
  boundary correctly). Add a handful of companion
  `UNarrowFrom::u_try_narrow` direct-call assertions so the
  own-trait `Outcome` return is also covered.

## Allowed escape hatches (not in scope — no change)

Per `mock/agent/rules/type-surface.md.tmpl`:

- `impl TryFrom<UFixed<...>> for UFixed<...> { type Error; fn try_from(...)
  -> Result<Self, Self::Error> }` — std trait method; signature fixed
  by std. Body will convert `Outcome` → `Result` at the boundary via
  a match, since notko does not ship `From<Outcome> for Result`
  today and we do not want to introduce that dependency here.
- `impl Iterator for ...::Iter { fn next(&mut self) -> Option<USize> }` —
  std trait, allowed.
- `impl PartialOrd for ... { fn partial_cmp(...) -> Option<Ordering> }`
  — std trait, allowed.
- `const NAME: &'static str` on `Strategy` — explicitly permitted
  (`&'static str` is allowed per CLAUDE.md primitive table).

## Raw-numeric const-generic accessors — `allow-bare-numeric`

Three `pub const fn` helpers in `arvo::strategy` intentionally return
raw `u8` / `usize`:

- `ibits_u8(i: IBits) -> u8`
- `fbits_u8(f: FBits) -> u8`
- `is_fractional(f: FBits) -> usize`

Rationale (already in doc comments): they exist so `I` / `F`
newtypes can appear inside anonymous const-generic expressions
where direct field access (`I.0`) is not permitted on the current
nightly. Removing them would block `UFixed<I, F, S>`'s use as a
const-generic argument anywhere.

Resolution: keep. Annotate each with
`// allow-bare-numeric: tracked: #71`. BACKLOG note capturing that
a dedicated `Bits` newtype with `const fn as_u8(self) -> u8` could
collapse both helpers once nightly allows more struct construction
inside const args — no timeline.

## `Option`/`Result`-flavoured `.map` / `Some` / `None` in bodies

The sweep reaches public signatures. Internal `Option::Some`/`None`
uses (e.g., `best = Some((i, d))` inside `min_degree_unvisited`)
become `Maybe::Is` / `Maybe::Isnt` as part of the same change,
because the return type change forces the match shape anyway.

## TryFrom boundary convention

For the two `impl TryFrom` sites in `ufixed.rs` / `ifixed.rs`, the
body becomes:

```rust
match <Hot as UNarrowFrom<Warm, _>>::u_try_narrow(src.to_raw()) {
    Outcome::Ok(v) => Ok(Self::from_raw(v)),
    Outcome::Err(()) => Err(()),
}
```

Pattern captured once in DOC; re-used verbatim across all four
TryFrom impls. This is the established cross-vocabulary adapter
shape: manual match at the std boundary, no `From` conversion
introduced in notko.
