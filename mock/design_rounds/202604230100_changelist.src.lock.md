## Deliverables

Rust source under `mock/crates/arvo-sparse/src/` implementing:

- `Csr<const ROWS: usize, const NNZ: usize, W: Copy>` ‚Äî `row_ptr: [USize; ROWS]`, `col_idx: [NodeId; NNZ]`, `values: [W; NNZ]`. Constructor `new()` = all-zero / empty. Query `get(row, col) -> Option<W>`, `row_values(row) -> &[W]`, `row_col_indices(row) -> &[NodeId]`, `nnz(row) -> USize`. No SpMV this round (backlog).
- `rcm_reorder<const N: usize>(adjacency: &BitMatrix64<N>) -> [NodeId; N]` ‚Äî Reverse Cuthill-McKee. Pick a pseudo-peripheral start (min-degree node), BFS ordering, reverse the result. Inserts `NodeId::new(USize(i))` at position `new_pos`.
- `block_diagonal<const N: usize>(adjacency: &BitMatrix64<N>) -> (USize, [USize; N])` ‚Äî find connected components on the adjacency's bipartite-like structure (delegate to arvo-graph's `components` since the computation is identical when treating the matrix as a graph). Returns `(block_count, block_id_per_node)`.
- `DulmageMendelsohn<const N: usize> { horizontal: Mask64, vertical: Mask64, square: Mask64 }` + `dulmage_mendelsohn<const N: usize>(adjacency: &BitMatrix64<N>) -> DulmageMendelsohn<N>` ‚Äî classify rows/columns by presence/absence of incoming/outgoing edges. `horizontal` = rows whose successors() is empty (dead-end output). `vertical` = "columns" that are sinks, i.e. no row has them in its successors (read-only input). `square` = core matched nodes.
- Unit tests: Csr round-trip, RCM on a known bandwidth-matters DAG, block_diagonal on a disconnected adjacency, Dulmage-Mendelsohn classification on a known-shape matrix.

Out of scope:

- `BitMatrix256` / `Mask256` variants (backlog).
- SpMV (backlog).
- Pseudo-peripheral finder beyond min-degree heuristic ‚Äî simpler heuristic is adequate for round scope.

## Impl-time decisions (pre-committed)

1. **Csr stays minimal.** Query-side accessors + construction only. Mutation API (`insert`, `remove`) is deferred ‚Äî downstream consumers typically construct Csr once from a dense matrix and read it afterwards.

2. **`block_diagonal` reimplements DFS locally.** arvo-sparse's forbidden-imports lint config prohibits `arvo_graph::*`. Rather than loosen the lint, reimplement a small iterative DFS inside `block.rs`. Saves cross-crate coupling.

3. **`rcm_reorder` start node.** Min-degree heuristic: pick the node with fewest non-zero successors+predecessors. Tie-break by lowest `NodeId`. Simple and good enough; pseudo-peripheral via BFS-diameter is a benchmark-round optimization.

4. **`dulmage_mendelsohn` simple split.** Classify each node i:
   - `horizontal`: `adjacency.successors(i).is_empty() && !adjacency.predecessors(i).is_empty()` ‚Üí row reads but never writes (dead-end output).
   - `vertical`: `adjacency.predecessors(i).is_empty() && !adjacency.successors(i).is_empty()` ‚Üí source node, pure-producer.
   - `square`: everything else that has both inputs and outputs (the core).
   - Nodes with no edges at all (isolated) are classified as `vertical` (treat as read-only constants).

5. **No `#[inline]` this round.** Benchmark optimization is its own future round.
