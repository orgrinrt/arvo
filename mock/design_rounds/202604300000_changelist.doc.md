### Section touching `Bits<N>` (if present; otherwise add under the
"Const generic bit-count types" / "Traits" area as a new subsection)

Replace the current description with the corrected definition:

- `Bits<N>` is an opaque N-bit pattern backed by the smallest native
  container that fits N bits: u8 for 1..=8, u16 for 9..=16, u32 for
  17..=32, u64 for 33..=64.
- Container selection is via `BitContainerFor<const N: u8>` â€” a
  sealed trait in arvo-bits mirroring `UContainerFor` in arvo.
- `#[repr(transparent)]` over the container; `size_of::<Bits<N>>()`
  matches the container size (1, 2, 4, or 8 bytes).
- `Bits::new(u64) -> Self` and `Bits::bits(self) -> u64` keep u64
  signatures at the API level; internal narrowing / widening is
  mechanical and masked to N bits.
- Use cases: opaque identity types (hash digests, packed flag
  containers, handle IDs). For an N-bit ARITHMETIC value, reach for
  `UFixed<{IBits(N)}, {FBits::ZERO}, Hot>` or a named alias
  (`Byte`, `Word`, `DWord`, `QWord`) instead.

### Add a new section "Bitfield â€” packed layouts with named sub-ranges"

Documents the `bitfield!` macro. Example:

```rust
use arvo_bits::bitfield;

bitfield! {
    pub struct StrHandle: 32 {
        origin: 1 at 31,
        reserved: 3 at 28..=30,
        id: 28 at 0..=27,
    }
}
```

Generated `StrHandle` is `#[repr(transparent)]` over `<BitContainer
as BitContainerFor<32>>::T` = u32, i.e. 4 bytes. Each named sub-
range gets `fn <name>(self) -> <smallest-fit-primitive>` and `fn
with_<name>(self, value) -> Self`. Compile-time checks: sub-ranges
disjoint, all fit within N, N â‰¤ 64, no overlaps. Setters mask input
to the sub-range width.

No arithmetic. No sign. Peer to `Bits<N>`; the structural layer on
top of the opaque-N-bits container.
