## Why drift

`no-bare-static-str` (new lint from mockspace-hilavitkutin-stack-lints task #125): pre-hilavitkutin-str crates must gate every `const NAME: &str` / `static NAME: &str` behind `#[cfg(debug_assertions)]`. Static strings occupy `.rodata` in release builds; arvo is upstream of hilavitkutin-str and cannot intern via `Str` without introducing a dependency cycle.

## Call-site audit

Only `tests/cross_width.rs` uses `::NAME` (lines 20, 25, 30, 35, 40 — all test assertions). No release-path consumer; `RANK: u8` carries strategy identity for cross-width resolution.

## Decision

Wrap `NAME` in `#[cfg(debug_assertions)]` on the trait item and all four impls. Release builds drop the strings entirely. Tests compile under `cfg(test)` which implies `debug_assertions`, so assertions still resolve.

Rejected alternative — inline `lint:allow`: the drift is real (zero-cost substrate stance; static strings do not exist in release builds of this stack) and debug-gating removes the drift rather than silencing it.

## Out of scope

- Promoting `Strategy::NAME` to an interned `hilavitkutin_str::Str` handle (would require reversing the dependency direction — arvo is L0, below hilavitkutin-str).
- Renaming `RANK` / changing its role. It stays the release-path identity.
