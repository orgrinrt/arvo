`UContainerFor`:

```rust
pub trait BitContainerFor<const N: u8> {
    type T: Copy + Clone + PartialEq + Eq + PartialOrd + Ord + Default
          + core::hash::Hash + core::fmt::Debug + 'static;
}

pub struct BitContainer;

// Macro-generated per-N impls, following arvo/src/strategy.rs pattern.

    ($ty:ty, $($n:literal),+) => {
        $( impl BitContainerFor<$n> for BitContainer { type T = $ty; } )+
```

## `mock/crates/arvo-bits/src/bits.rs` — rewrite

Replace the current `pub struct Bits<const N: u8>(u64)` with the
container-dispatched version:

```rust
use crate::container::{BitContainer, BitContainerFor};

#[repr(transparent)]
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Default)]
pub struct Bits<const N: u8>(<BitContainer as BitContainerFor<N>>::T)
where
    BitContainer: BitContainerFor<N>;

impl<const N: u8> Bits<N>
where
    BitContainer: BitContainerFor<N>,
{
    const _BOUNDS: () = {
        assert!(N > 0, "Bits<0> is invalid; use Bool for 1-bit values");
        assert!(N <= 64, "Bits<N> requires N <= 64");
    };

    /// Construct a Bits<N> from raw bits, masking to the low N bits.
    /// Input is u64 (widest representable raw value); internal
    /// storage narrows to the container dispatched for N.
    pub fn new(raw: u64) -> Self {
        let _ = Self::_BOUNDS;
        let masked = raw & mask_u64(N);
        Self(narrow_to_container::<N>(masked))
    }

    /// The underlying bit pattern, widened to u64.
    pub fn bits(self) -> u64 {
        widen_to_u64::<N>(self.0)
    }
}
```

`mask_u64(N)` stays a private helper returning `(1u64 << N) - 1`
or `u64::MAX` for N=64.

`narrow_to_container::<N>(u64) -> <_ as BitContainerFor<N>>::T` and
`widen_to_u64::<N>(<_ as BitContainerFor<N>>::T) -> u64` are private
helpers implemented via four trait methods on a sealed
`NarrowWiden<N>` helper trait (one impl per container variant) or
via `const_cast!`-style match in a const-generic-if. Simplest:
four explicit `impl` blocks parameterised by the container type
via a second sealed trait. Concrete shape TBD at implementation
time; both helpers are const fns.

`new` stops being `const fn` only if the narrowing helper can't be
const. If it can be const, keep the const qualifier (preferable;
`Bits::new(...)` is used in trait const contexts).

`BitWidth` and `BitAccess` impls stay — update their internal
indexing to work via `widen_to_u64` where they currently read
`self.0` directly as u64.

## `mock/crates/arvo-bits/src/bitfield.rs` (NEW)

Declarative macro `bitfield!` that generates a `#[repr(transparent)]`
struct over `<BitContainer as BitContainerFor<N>>::T` with named
sub-range accessors.

Grammar:

```
bitfield! {
    $(#[$struct_attr:meta])*
    $vis:vis struct $name:ident: $n:literal {
        $(
            $(#[$field_attr:meta])*
            $field:ident: $field_bits:literal at $bit_spec:tt
        ),* $(,)?
    }
}
```

where `$bit_spec` is either a single bit `$idx:literal` or a range
`$lo:literal..=$hi:literal`.

Generates:

- `#[repr(transparent)] pub struct $name(<BitContainer as
  BitContainerFor<$n>>::T) where BitContainer: BitContainerFor<$n>;`
- Derives `Copy, Clone, Debug, PartialEq, Eq, Hash, Default`.
- `pub const BITS: u8 = $n;`
- `pub const fn new() -> Self` — zero-initialised.
- `pub const fn from_bits(raw: u64) -> Self` — masks to N bits,
  narrows to container.
- `pub const fn to_bits(self) -> u64` — widens container to u64.
- Per field:
  - `pub const fn $field(self) -> <smallest fit primitive>` — shifts
    right and masks to the field width.
  - `pub const fn with_$field(self, value: <smallest fit primitive>)
    -> Self` — masks value to field width, clears the field's bits
    in self, ORs in the shifted value.

Compile-time validation:

- Before generating accessors, the macro emits `const _: () = {
  assert!(<overlap-check>); };` blocks that pair every (i, j) of
  sub-ranges and fail if they share any bit.
- A further `assert!` enforces `hi < $n` for every sub-range.
- A further `assert!` enforces `$n <= 64`.

The compile-time checks live inside const blocks so their failure
produces standard "evaluation of constant value failed" errors at
the definition site.

## `mock/crates/arvo-bits/tests/bitfield.rs` (NEW)

Tests:

1. Layout — define a test struct `StrHandle: 32 { origin:1 at 31,
   reserved:3 at 28..=30, id:28 at 0..=27 }`. Assert
   `size_of::<StrHandle>() == 4`.
2. Round-trip — `new().with_origin(1).with_id(0x1234)` recovers the
   same sub-range values from the accessors.
3. Independence — setting one sub-range does not perturb others.
4. Masking — passing `value` wider than the sub-range truncates; no
   overflow into neighbours.
5. `from_bits` / `to_bits` — round-trips raw u64 through the
   container.
