# arvo-spectral — Design

Spectral graph methods on const-generic fixed-size structures.
`#![no_std]`, no alloc. Operates on dense weighted matrices via
`arvo_tensor::Matrix<W, N>`; does NOT depend on arvo-sparse at
runtime — small-N spectral methods fit in stack-allocated dense
arrays.

## Matrix wrapper

Dense `N × N` matrix storage uses `arvo_tensor::Matrix<W, N>` —
`#[repr(transparent)]` over `[[W; cap_size(N)]; cap_size(N)]` with
`get(i: USize, j: USize) -> W` / `set(i: USize, j: USize, v: W)` /
`from_fn(|r, c| -> W)` / `diagonal()`. Previously shipped local to
arvo-spectral; promoted to arvo-tensor under round 202604280000
(arvo-comb sweep) for shared use across the algorithm crates.
arvo-spectral re-exports:

```rust
pub use arvo_tensor::Matrix;
```

so existing consumers keep their `arvo_spectral::Matrix` import
paths.

Generic over numeric traits for weights and eigenvectors. Weight
matrices use trait-bounded types; eigenvectors use float-like types.
Weight-to-float conversion happens at the API boundary via `Into<F>`.
Consumer instantiates with concrete arvo types; this crate is library
code.

## Capacity

Ships with `Mask64`-backed partition outputs (supports N ≤ 64). For
larger problems, a `Mask256`-backed sibling set of functions lives in
`BACKLOG.md.tmpl`. Spectral methods are typically used at small N
(30-ish iterations, N ≤ 32 per polka-dots research), so `N ≤ 64` is
not a practical limit for the common case.

## Laplacian construction

Build the graph Laplacian `L = D - W` from a weighted adjacency
matrix. `D` is the diagonal degree matrix, `W` is the weighted
adjacency. Input and output use the same matrix format from
arvo-sparse.

## Power iteration

Compute the dominant eigenvector of a matrix via iterative
multiplication. Const-generic stack-allocated: `[[W; N]; N]` weight
matrix and `[F; N]` eigenvector. Convergence after a fixed iteration
count (tunable const generic, typically ~30 iterations).

Stack budget at N=64: weight matrix = N x N x size_of(W), eigenvector
= N x size_of(F). Exact sizes depend on consumer's type choices.
Microseconds at pipeline scale. Zero heap.

## Fiedler vector

The eigenvector corresponding to the second-smallest eigenvalue of
the Laplacian. Computed via power iteration on the deflated Laplacian
(subtract the projection onto the constant eigenvector).

The Fiedler vector's sign partitions the graph: positive entries in
one group, negative in another. This bisection minimizes the cut
weight — the total edge weight crossing the partition boundary.

## Spectral bisection

```rust
fn spectral_bisection<const N: Cap, F>(
    fiedler: &[F; N.0.0 as usize],
) -> (Mask64, Mask64)
where
    F: TotalOrd + Copy + FromConstant
```

Partition nodes into two groups by the sign of the Fiedler vector:
positive entries go into the first returned mask, non-positive into
the second. The cut weight (sum of edge weights between groups) is
minimized by construction of the Fiedler vector. Requires N ≤ 64 this
round (Mask64 output).

Why spectral beats greedy: spectral considers the entire graph
structure globally. Greedy partitioning finds locally optimal
solutions that can miss the global minimum cut.

## k-way partitioning

```rust
fn k_way_partition<const N: Cap, const K: Cap, W, F>(
    weights: &Matrix<W, N>,
    iterations: USize,
) -> (USize, [USize; N.0.0 as usize])
where
    W: Into<F> + Copy,
    F: Add<Output = F>
     + Sub<Output = F>
     + Mul<Output = F>
     + Sqrt
     + Recip
     + TotalOrd
     + Copy
     + FromConstant
```

For more than 2 groups: recursive bisection. Apply spectral bisection
to get 2 groups, then recursively bisect each group until K groups or
a group cannot be usefully split. Returns `(actual_partition_count,
partition_id_per_node)`; `actual_partition_count` may be less than K.

Used when the consumer needs minimum-cut grouping of a weighted
graph at a scale where greedy bisection is insufficient. Researched
threshold: engage spectral when the grouping problem has more than
5 nodes; below that a single trunk suffices (polka-dots T3 decision
#69). Power iteration converges in ~30 iterations at pipeline scale.

## Consumers

Typical usage: build a weighted graph where nodes are groupable
units and edge weights represent shared work / data volume between
them. Fiedler bisection cuts along the boundary with minimum total
shared work. Consumer supplies concrete numeric types: weights as
arvo fixed-point, eigenvectors as the `Float` alias (FastFloat /
StrictFloat via cfg).
